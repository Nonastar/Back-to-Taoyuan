# 钓鱼小游戏修复记录

> **创建日期**: 2026-04-11
> **系统**: M01 钓鱼小游戏
> **状态**: 已修复

## 问题汇总

实现过程中发现设计文档与实际可玩性存在差距，主要集中在时机小游戏阶段。以下是发现的问题及修复方案。

---

## 1. 时机窗口时间过短

### 问题描述
设计文档中的时机窗口公式 `0.3s + (难度 × 0.05s)` 导致低难度鱼的窗口时间极短。

### 原设计参数
```
基础窗口时间 = 0.3 秒
鱼类难度加成 = fish.difficulty × 0.05 秒
(难度 1-10 → 加成 0.05-0.5 秒)
最大值限制: 0.8 秒
```

### 问题现象
```
timing window: 0.35s  ← 难度1鱼的时机窗口
timing_result: "too_late"  ← 玩家还没反应过来就过期了
```

### 修复参数
```
基础窗口时间 = 1.0 秒
鱼类难度加成 = fish.difficulty × 0.1 秒
(难度 1-10 → 加成 0.1-1.0 秒)
最大值限制: 2.0 秒
```

### 修复后时机窗口
| 难度 | 原窗口 | 修复窗口 |
|------|--------|----------|
| 1 (蓝鳃鱼) | 0.35s | **1.1s** |
| 5 (中等) | 0.55s | **1.5s** |
| 10 (传说) | 0.8s | **2.0s** |

---

## 2. 浮标下沉速度过快

### 问题描述
原实现中浮标下沉速度为 `delta * 2.0`，导致浮标约0.5秒内完全消失。

### 问题现象
```
Timing window started! duration=0.7s
Timing window ended              ← 0.7秒后窗口结束
Reel pressed!                   ← 玩家才反应过来
timing_result: "too_late"       ← 已过期
```

### 问题分析
- 时机窗口持续0.7秒
- 但浮标在0.5秒内就完全消失
- 玩家看到"鱼上钩"提示时，浮标已经消失

### 修复方案
```
浮标下沉速度 = delta / 2.5  (2.5秒从0到100%)
```

### 修复后时机流程（难度1）
```
0.0s   鱼上钩，浮标开始下沉
1.75s  浮标下沉到70%，时机窗口开始（1.1秒）
2.5s   浮标完全消失
2.85s  时机窗口结束

玩家有 1.75s ~ 2.85s 的反应时间（约1.1秒）
```

## 3. 时机窗口触发时机错误

### 问题描述
原实现中时机窗口在进入 BITE 阶段时立即触发，而不是在下沉超过70%时触发。

### 原实现逻辑
```
BITE阶段开始 → _is_timing_window_active = true → 立即开始计时
```

### 修复实现
```
浮标下沉超过70%时 → _is_timing_window_active = true → 开始计时
```

### 设计文档依据
> **窗口触发**: 浮标下沉超过 70%

## 4. 时机判定逻辑过严

### 问题描述
时机窗口结束后、浮标完全消失前提竿，直接判定为"太晚"失败，没有给予玩家容错空间。

### 原判定逻辑
```
if 时机窗口激活:
    → 完美时机
elif 浮标已消失:
    → 太晚失败
else:
    → 太晚失败
```

### 问题分析
时机窗口结束（0.7秒）后，浮标可能还未完全消失（需要2.5秒）。这段时间（约1.8秒）玩家提竿应该仍有机会进入搏鱼阶段。

### 修复判定逻辑
```
if 时机窗口激活:
    → 完美时机 → 进入搏鱼
elif 浮标消失（DIVING）:
    → 太晚失败
elif 浮标下沉中（SINKING）但时机窗口已过:
    → 正常时机 → 进入搏鱼（有惩罚）
else:
    → 太早失败
```

### 修复依据
给予玩家约0.75秒的容错时间（时机窗口结束到浮标完全消失），在此期间提竿可进入搏鱼阶段但判定为"正常时机"。

## 5. 鱼冲刺行为与设计不符

### 设计文档描述
> 鱼会随机向左右两侧冲刺

### 问题
"随机冲刺"没有体现鱼试图"逃离"的特性。

### 修复实现
```
70%概率：向逃离方向冲刺
  - 鱼在中心左侧(<0.5) → 逃离方向为左(-1)
  - 鱼在中心右侧(>0.5) → 逃离方向为右(+1)
30%概率：随机方向冲刺
```

### 自然拉力
```
无输入时：鱼趋向危险区
  - 鱼在左侧 → 继续向左
  - 鱼在右侧 → 继续向右
```

## 6. GDScript 语法错误

### 问题1: `_pass` vs `pass`
```gdscript
# 错误
State.IDLE:
    _pass

# 正确
State.IDLE:
    pass
```

### 问题2: 场景文件注释
```gdscript
# 错误 - .tscn 文件中不能使用 # 注释
[node name="Foo" type="Node"]
# 这是一个注释

# 正确 - 移除注释
[node name="Foo" type="Node"]
```

### 问题3: TextureProgressBar vs ProgressBar
```gdscript
# 错误 - TextureProgressBar 需要配置纹理
var _bobber_bar: TextureProgressBar

# 正确 - 使用 ProgressBar 更简单
var _bobber_bar: ProgressBar
```
