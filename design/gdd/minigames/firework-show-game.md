# 烟花秀小游戏 (Firework Show Game)

> **状态**: In Design
> **Author**: Claude Code
> **Last Updated**: 2026-04-07
> **System ID**: M09
> **Implements Pillar**: Mini-game 玩法
> **Festival**: 国庆节 (National Day) / 春节

## Overview

烟花秀小游戏是国庆节节日活动中的创意表演玩法。玩家通过发射烟花，在夜空中创造美丽的图案和造型。

游戏采用创意表现模式：
- **发射烟花**：控制发射角度和时机
- **图案绘制**：引导烟花形成指定图案
- **自由创作**：自由模式下创造独特造型

每场游戏有5次发射机会，根据图案完成度和创意表现获得积分。

小游戏独立于主游戏运行，完成后根据成绩与活动系统交互进行奖励结算。

## Player Fantasy

烟花秀小游戏给玩家带来**视觉盛宴的震撼和创意表达的满足感**。玩家应该感受到：

- **升空的期待** — 看着烟花冲上天空的激动
- **绽放的绚烂** — 烟花在夜空中绽放出璀璨的光芒
- **图案的惊喜** — 引导烟花形成美丽图案的成就感
- **色彩的变幻** — 欣赏五彩缤纷的烟花表演
- **国庆的骄傲** — 感受节日庆典的欢乐氛围

参考游戏：
- **Patapon**: 节奏指挥的韵律感
- **Fireworks Simulator**: 烟花模拟的视觉效果
- **Sky Guide**: 星空观测的宁静感

**情感曲线**:
1. **开场**: 夜空场景，国庆装饰，庆典氛围
2. **前期**: 熟悉发射操作，感受烟花绽放
3. **中期**: 尝试引导图案，掌握技巧
4. **后期**: 追求完美图案，挑战高评分
5. **欣赏**: 观看完整烟花秀，分享喜悦

**与国庆节的关联**:
- 国庆节是举国欢庆的日子，烟花表演是庆典高潮
- 夜空中的璀璨光芒象征祖国的繁荣昌盛
- 背景音乐可以是气势磅礴的交响乐

## Detailed Design

### Core Rules

#### 1. 游戏设置

```
发射次数: 5次
游戏时长: 每发射后等待烟花绽放+冷却约5秒
图案目标: 屏幕显示目标图案，玩家需引导烟花形成
```

#### 2. 操作两要素

##### 2.1 发射角度

```
操作: 方向键/摇杆调整发射角度
角度范围: 45° 到 135° (从地平线算起)

瞄准辅助:
- 显示发射轨迹预测线
- 显示目标图案的大致位置
- 帮助玩家判断发射方向
```

##### 2.2 发射时机

```
时机判定:
- 烟花在上升过程中可微调方向
- 到达顶点前松开，进入绽放阶段
- 时机影响烟花初始位置

时机影响:
- 最佳时机发射: 图案位置更准确
- 提前/延后发射: 图案位置偏移
```

#### 3. 烟花类型

##### 3.1 基础烟花

| 类型 | 颜色 | 爆炸范围 | 持续时间 | 基础分 |
|------|------|----------|----------|--------|
| **圆形** | 红色 | 大 | 2秒 | 50分 |
| **星形** | 金色 | 中 | 1.5秒 | 60分 |
| **心形** | 粉色 | 小 | 1秒 | 80分 |
| **环形** | 蓝色 | 中 | 2秒 | 70分 |
| **瀑布** | 白色 | 大 | 3秒 | 90分 |

##### 3.2 特殊烟花

| 类型 | 颜色 | 效果 | 基础分 |
|------|------|------|--------|
| **彩虹** | 七彩 | 连续多种颜色渐变 | 150分 |
| **双子** | 双色 | 两次连续爆炸 | 120分 |
| **彗星** | 拖尾 | 长距离拖尾效果 | 100分 |

#### 4. 图案系统

##### 4.1 图案难度

| 难度 | 图案复杂度 | 完成阈值 | 基础分 |
|------|------------|----------|--------|
| **简单** | 圆形、正方形 | 70% | 100分 |
| **中等** | 星星、爱心 | 60% | 150分 |
| **困难** | 文字、复杂图形 | 50% | 200分 |

##### 4.2 图案完成度计算

```
图案完成度 = 正确位置烟花数 / 目标位置总数

正确位置定义:
- 烟花爆炸点距离目标位置 < 图案容差
- 容差范围: 图案半径的 20%

完成度等级:
| 完成度 | 评价 | 得分系数 |
|--------|------|----------|
| ≥90% | 完美 | 1.0 |
| ≥70% | 优秀 | 0.8 |
| ≥50% | 良好 | 0.6 |
| <50% | 一般 | 0.4 |
```

#### 5. 连击系统

```
连击计数: 连续图案完成度≥50%时连击数+1
连击中断: 完成度<50%时重置为0

连击奖励:
| 连击数 | 额外加分 |
|--------|----------|
| 2连 | +30分 |
| 3连 | +60分 |
| 4连 | +100分 |
| 5连 | +200分 |
```

#### 6. 奖励结算

##### 6.1 分数段奖励

| 分数段 | 称号 | 金币奖励 |
|--------|------|----------|
| ≥1500分 | 烟花大师 | 500g |
| ≥1200分 | 烟花高手 | 300g |
| ≥800分 | 烟花新秀 | 150g |
| ≥400分 | 参与奖 | 50g |

##### 6.2 额外奖励

| 条件 | 额外奖励 |
|------|----------|
| 首次获得"烟花大师" | 成就解锁 |
| 全部图案完美(≥90%) | +200g |
| 刷新个人最高分 | +100g |

#### 7. 每日限制

```
每日参与次数: 3次
每日重置: 每日凌晨6点
超出限制: 提示"今日次数已用完"
```

## Formulas

### 1. 图案完成度计算

```
图案完成度 = (正确位置烟花数 / 目标位置总数) × 100%
```

**变量定义**:

| 变量 | 含义 | 范围 |
|------|------|------|
| `correct_shots` | 正确位置的烟花数量 | 0 ~ target_positions |
| `target_positions` | 目标图案需要的总位置数 | 根据图案复杂度变化 |
| `PATTERN_TOLERANCE` | 位置容差（图案半径的百分比） | 默认 20% |
| `max_distance` | 判定正确的最大距离 | tolerance_radius |

**判定逻辑**:

```
is_correct_position(explosion_point, target_point, tolerance) {
    distance = euclidean_distance(explosion_point, target_point)
    return distance <= tolerance
}
```

### 2. 单次发射得分计算

```
发射得分 = 基础分 × 图案系数 × 时机系数 × 连击系数
```

**变量定义**:

| 变量 | 含义 | 默认值 |
|------|------|--------|
| `BASE_SCORE` | 基础分（根据烟花类型） | 见烟花类型表 |
| `pattern_coefficient` | 图案完成度系数 | 0.4 ~ 1.0 |
| `timing_coefficient` | 时机系数 | 0.8 ~ 1.2 |
| `combo_multiplier` | 连击加成系数 | 1.0 ~ 1.5 |

**图案系数映射**:

| 完成度 | 评价 | 图案系数 |
|--------|------|----------|
| ≥90% | 完美 | 1.0 |
| ≥70% | 优秀 | 0.8 |
| ≥50% | 良好 | 0.6 |
| <50% | 一般 | 0.4 |

**时机系数**:

| 时机判定 | 系数 |
|----------|------|
| 最佳时机（顶点±5°） | 1.2 |
| 良好时机（顶点±15°） | 1.0 |
| 一般时机（顶点±30°） | 0.8 |

**时机计算**:

```
时机判定角度 = abs(当前角度 - 顶点角度)
if 时机角度 <= 5: timing_coefficient = 1.2
elif 时机角度 <= 15: timing_coefficient = 1.0
elif 时机角度 <= 30: timing_coefficient = 0.8
else: timing_coefficient = 0.8
```

### 3. 连击加成计算

```
连击奖励分 = COMBO_BONUS[当前连击数]
```

**连击加成表**:

| 连击数 | 额外加分 |
|--------|----------|
| 2连 | +30分 |
| 3连 | +60分 |
| 4连 | +100分 |
| 5连 | +200分 |
| >5连 | +200分（维持最大） |

**连击判定**:

```
if 图案完成度 >= 50%:
    combo_count += 1
    combo_bonus = COMBO_BONUS[combo_count]
else:
    combo_count = 0
    combo_bonus = 0
```

### 4. 最终得分计算

```
最终得分 = Σ(单次发射得分) + Σ(连击奖励) + 额外奖励
```

**额外奖励**:

| 条件 | 额外奖励 |
|------|----------|
| 全部图案完美(≥90%) | +200g |
| 刷新个人最高分 | +100g |
| 首次获得"烟花大师" | 成就解锁 |

### 5. 烟花基础分表

**基础烟花**:

| 烟花类型 | 基础分 |
|----------|--------|
| 圆形 | 50分 |
| 星形 | 60分 |
| 心形 | 80分 |
| 环形 | 70分 |
| 瀑布 | 90分 |

**特殊烟花**:

| 烟花类型 | 基础分 |
|----------|--------|
| 彩虹 | 150分 |

### 6. 发射轨迹预测

```
预测高度(角度) = initial_velocity² × sin²(angle) / (2 × gravity)
预测距离(角度) = initial_velocity² × sin(2 × angle) / gravity
```

**发射参数**:

| 参数 | 值 | 说明 |
|------|-----|------|
| `INITIAL_VELOCITY` | 50 m/s | 发射初速度 |
| `GRAVITY` | 9.8 m/s² | 重力加速度 |
| `MIN_ANGLE` | 45° | 最小发射角度 |
| `MAX_ANGLE` | 135° | 最大发射角度 |

### 7. 称号判定

```

```

| 分数段 | 称号 | 金币奖励 |
|--------|------|----------|
| ≥1500分 | 烟花大师 | 500g |
| ≥1200分 | 烟花高手 | 300g |
| ≥800分 | 烟花新秀 | 150g |
| ≥400分 | 参与奖 | 50g |
| <400分 | 鼓励奖 | 0g |

### 8. 示例计算

**场景**: 玩家发射第3发
- 烟花类型: 星形（基础分60）
- 图案完成度: 75%（优秀，系数0.8）
- 时机判定: 良好（系数1.0）
- 当前连击: 2连（加成+30分）
- 连击系数: 1.2

**计算**:
```
单次得分 = 60 × 0.8 × 1.0 × 1.2 = 57.6 ≈ 58分
连击奖励 = 30分
本发射总得分 = 58 + 30 = 88分
```

## Edge Cases

### 1. 游戏状态异常

#### 1.1 发射过程中游戏中断

```
场景: 玩家在发射升空过程中强制退出游戏
处理:
  - 保存当前发射状态
  - 如果爆炸已完成，计入得分
  - 如果爆炸未完成，该发射不计入
  - 重新进入时从当前状态继续或重置
```

#### 1.2 5次发射全部未完成

```
场景: 玩家在发射过程中全部取消或失败
处理:
  - 每发有最小得分（完成度<50%，基础分×0.4）
  - 即使全部取消，也显示结果
  - 称号判定: <400分 → 鼓励奖
```

#### 1.3 同时满足多个称号条件

```
场景: 分数同时满足多个段位
处理:
  - 取最高称号（如1500分同时满足"烟花大师"和"烟花高手"）
  - 只发放最高段位奖励
```

### 2. 图案判定异常

#### 2.1 图案点过于密集

```
场景: 目标图案有重叠的点
处理:
  - 同一位置的烟花可同时满足多个目标点
  - 每个目标点只能被一个烟花满足
  - 判定顺序: 距离从近到远
```

#### 2.2 烟花爆炸在边界

```
场景: 烟花在屏幕边缘爆炸
处理:
  - 屏幕边界内的部分计入判定
  - 超出边界的部分不计入
  - 提示"烟花超出显示范围"
```

#### 2.3 多个烟花同时爆炸

```
场景: 双子烟花等特殊烟花多次爆炸
处理:
  - 每次爆炸独立计算位置
  - 分别计入图案完成度
  - 全部爆炸完成后统一计分
```

### 3. 连击系统异常

#### 3.1 第一发失败后第二发完美

```
场景: 第一发完成度<50%，第二发≥90%
处理:
  - 连击从第二发重新计算
  - 第二发计为连击起点（combo=1）
  - 不继承第一发的状态
```

#### 3.2 最后一发与连击判定

```
场景: 第五发（最后一发）后连击达到5连
处理:
  - 第五发计入连击
  - 显示"5连达成！"提示
  - 连击奖励加入总分
  - 游戏直接进入Showtime
```

### 4. 时间相关异常

#### 4.1 烟花在空中时时间到期

```
场景: 烟花在升空或爆炸过程中达到预设时间
处理:
  - 强制触发爆炸
  - 位置按当前轨迹计算
  - 图案完成度正常计算
```

#### 4.2 每日重置恰好在游戏中

```
场景: 每日凌晨6点重置时玩家正在进行游戏
处理:
  - 当前游戏局不计入当日次数
  - 完成后显示"今日次数已用完"
  - 不影响已完成局数的奖励
```

### 5. 网络/存档异常

#### 5.1 奖励结算时网络断开

```
场景: 游戏完成但奖励结算时网络断开
处理:
  - 分数和称号本地保存
  - 奖励标记为"待领取"
  - 重新连接后补发奖励
  - 每日重置前未领取则补发
```

#### 5.2 本地存档损坏

```
场景: 最高分记录存档损坏
处理:
  - 检测到损坏时重置为0
  - 显示"记录已重置"
  - 不影响当前游戏
```

### 6. 特殊烟花异常

#### 6.1 彩虹烟花色彩同步

```
场景: 彩虹烟花的色彩渐变与音乐节拍不同步
处理:
  - 以游戏内计时器为准
  - 不依赖外部音频同步
  - 色彩变化固定为2秒7色
```

#### 6.2 彗星烟花拖尾超出屏幕

```
场景: 彗星烟花的长拖尾效果超出显示范围
处理:
  - 拖尾在屏幕边界截断
  - 不影响爆炸位置判定
  - 爆炸点正常计入图案
```

### 7. 用户操作异常

#### 7.1 快速连续点击发射

```
场景: 玩家在发射冷却中再次点击
处理:
  - 忽略点击
  - 显示"等待烟花绽放..."
  - 冷却时间固定为2秒
```

#### 7.2 角度调整超出范围

```
场景: 玩家尝试将角度调整到45°以下或135°以上
处理:
  - 角度锁定在有效范围内
  - 轻微震动反馈提示边界
```

#### 7.3 同时按多个方向键

```
场景: 左右方向键同时按下
处理:
  - 忽略冲突输入
  - 保持当前角度不变
  - 不触发发射
```

### 8. 性能相关异常

#### 8.1 烟花数量过多导致卡顿

```
场景: 图案要求大量烟花导致性能下降
处理:
  - 限制同时显示的烟花数量（最大50个粒子）
  - 降低低配设备的粒子效果
  - 保持计分逻辑完整
```

#### 8.2 特殊烟花效果与普通烟花重叠

```
场景: 多个烟花同时在相近位置爆炸
处理:
  - 使用叠加混合模式
  - 视觉上产生更亮效果
  - 独立计算各烟花得分
```

### 详细依赖说明

#### F01 时间/季节系统（硬依赖）

**为什么依赖**: M09 烟花秀是节日限定活动，只能在国庆节或春节期间参与。

**接口需求**:
- `is_national_day() -> bool` — 检查是否在国庆节期间
- `is_spring_festival() -> bool` — 检查是否在春节期间
- `get_festival_theme() -> String` — 获取节日主题用于 UI 配色

**被依赖内容**: 无（M09 是叶子节点）

#### F05 音效系统（硬依赖）

**为什么依赖**: 音效是烟花体验的核心组成部分。

**接口需求**:
- 发射音效、环境音乐、爆炸音效
- 连击成功音效、完美图案音效
- BGM 播放控制

**被依赖内容**: 无（M09 是叶子节点）

#### C01 玩家属性系统（软依赖）

**为什么依赖**: 奖励结算需要修改玩家金币。

**接口需求**:
- `add_currency(amount: int, source: String) -> bool` — 添加金币

**被依赖内容**: 无（M09 是叶子节点）

#### P08 任务系统（软依赖）

**为什么依赖**: 烟花秀是节日活动的组成部分，可能有相关任务。

**接口需求**:
- `on_minigame_complete(game_id: String, result: Dictionary)` — 任务回调

**被依赖内容**:
- P08 需要知道 M09 的完成条件

#### P09 成就系统（软依赖）

**为什么依赖**: 存在"烟花大师"等成就。

**接口需求**:
- `unlock_achievement(achievement_id: String) -> bool` — 解锁成就
- `increment_stat(stat_id: String, value: int) -> void` — 统计计数

**被依赖内容**:
- P09 需要知道 M09 的成就触发条件

### 8. 调优建议

**降低难度**:
- 提高 `PATTERN_TOLERANCE` 至 0.30
- 降低 `TIMING_PERFECT_ANGLE` 至 8°
- 降低 `COMBO_THRESHOLD` 至 0.40
- 降低称号阈值 20%

**提高难度**:
- 降低 `PATTERN_TOLERANCE` 至 0.15
- 提高 `TIMING_PERFECT_ANGLE` 至 3°
- 提高 `COMBO_THRESHOLD` 至 0.60
- 提高称号阈值 20%

**平衡测试重点**:
- 验证 5 连达成率 < 5%
- 验证大师称号获得率 < 10%
- 验证平均完成度在 50%~70%
