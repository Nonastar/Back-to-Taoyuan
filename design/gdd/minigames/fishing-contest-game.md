## Overview

钓鱼比赛小游戏是端午节节日活动中的竞技钓鱼玩法。玩家在规定时间内（3分钟）尽可能多地钓到鱼，并根据钓到的鱼类稀有度和数量进行排名。游戏利用M01钓鱼小游戏的核心机制，但在比赛模式下增加了时间限制、多人竞钓和特殊鱼种等要素。

设计为 Standard 复杂度，包含：
1. **计时钓鱼**：3分钟限时，尽可能多地钓到鱼
2. **鱼类计分**：不同稀有度鱼类获得不同积分
3. **特殊鱼种**：比赛中出现"幸运鱼"（双倍积分）和"炸弹鱼"（扣分）
4. **排行榜**：显示当前排名和历史最佳成绩

小游戏独立于主游戏运行，完成后根据名次与活动系统交互进行奖励结算。

## Player Fantasy

钓鱼比赛小游戏给玩家带来**争分夺秒的紧张感和收集稀有的兴奋感**。玩家应该感受到：

- **倒计时的压力** — 看着时间一秒秒流逝，必须抓紧每一竿
- **连竿的快感** — 连续钓到鱼类，积分飙升
- **幸运鱼的狂喜** — 双倍积分的幸运鱼上钩时的惊喜
- **炸弹鱼的懊恼** — 扣分的炸弹鱼打乱节奏
- **冲刺的紧迫** — 最后30秒，拼命钓鱼争排名

参考游戏：
- **钓鱼大赛模式**: 限时钓鱼竞技
- **鱼类收集游戏**: 不同稀有度的收集乐趣

**情感曲线**:
1. **开场准备**: 熟悉钓点，规划策略
2. **前期积累**: 稳扎稳打，积累基础分
3. **中期爆发**: 找到节奏，连竿得分
4. **后期冲刺**: 时间紧迫，全力以赴
5. **结算公布**: 紧张等待，排名揭晓

## Detailed Design

### Core Rules

#### 1. 比赛时间系统

##### 1.1 时间设置

```
比赛时长: 3 分钟 (180 秒)
倒计时显示: 屏幕顶部显示剩余时间
最后30秒: 时间显示变红，播放警报音
时间耗尽: 比赛结束，进入结算
```

##### 1.2 钓鱼周期

```
每次钓鱼流程:
  抛竿等待: 2-5 秒 (随机)
  咬钩判定: 根据鱼类难度
  提竿判定: M01 时机小游戏
  搏鱼过程: 1-60 秒 (根据鱼类难度和玩家操作)
  捕获结果: 显示鱼类和积分

平均每次钓鱼: 8-15 秒
预计比赛可钓鱼数: 12-22 条

> 注意：搏鱼时间完全取决于 M01 的搏鱼小游戏，玩家操作影响较大。
> M01 规定搏鱼最长 60 秒（超时则鱼逃跑）。
```

#### 2. 鱼类系统

##### 2.1 鱼类稀有度

| 稀有度 | 英文 | 基础分 | 出现概率 | 示例 |
|--------|------|--------|----------|------|
| **普通** | Common | 10分 | 45% | 鲫鱼、小龙虾 |
| **优质** | Good | 25分 | 25% | 鲤鱼、鳜鱼 |
| **精品** | Rare | 50分 | 15% | 金鱼、鳗鱼 |
| **传说** | Legendary | 100分 | 5% | 锦鲤、金龙鱼 |
| **幸运鱼** | Lucky | 双倍 | 5% | 任意鱼×2分 |
| **炸弹鱼** | Bomb | -50分 | 5% | 河豚(扣分) |

> **概率说明**：普通45% + 优质25% + 精品15% + 传说5% = 90%，剩余10%中幸运鱼5%、炸弹鱼5%。

##### 2.2 特殊鱼种

```
幸运鱼 (Lucky Fish):
  - 外观: 普通鱼但带有金色光晕
  - 效果: 本次钓鱼积分×2
  - 提示: "Lucky! Double Points!"
  - 注意: 幸运鱼与炸弹鱼互斥，不会同时触发

炸弹鱼 (Bomb Fish):
  - 外观: 河豚，带有红色警告
  - 效果: 扣除50分
  - 提示: "Bomb! -50 Points!"
  - 出现: 随机替换普通鱼
  - 注意: 炸弹鱼不享受速度加成

幸运炸弹鱼处理:
  - 幸运鱼和炸弹鱼通过独立的随机roll决定，互不叠加
  - 不会出现"幸运炸弹鱼"（既双倍又扣分的情况）
```

#### 3. 积分系统

##### 3.1 积分计算

```
单次钓鱼积分 = 基础分 × 稀有度倍率 × 幸运加成 × 速度加成

稀有度倍率:
  普通: ×1.0
  优质: ×2.5
  精品: ×5.0
  传说: ×10.0

幸运加成:
  非幸运鱼: ×1.0
  幸运鱼: ×2.0

速度加成:
  快速完成 (<5秒): ×1.2
  正常完成 (5-10秒): ×1.0
  慢速完成 (>10秒): ×0.8
```

##### 3.2 最低分限制

```
最低分: -50 (炸弹鱼扣分)
不会因为多次炸弹鱼变成负分

如果当前积分 + 本次得分 < 0:
    本次得分为 0 (不扣分)
```

#### 4. 排行榜系统

##### 4.1 排名规则

```
按总积分排名
同分时，按钓到鱼的总数排名
再相同，按传说鱼数量排名
```

##### 4.2 排行榜显示

```
实时排行榜: 比赛期间显示当前排名
历史最佳: 显示玩家历史最高分
```

#### 5. 比赛流程

## Formulas

### 1. 鱼类生成

```
# 鱼类出现概率 (与表格一致)
fish_roll = random(0.0, 1.0)

if fish_roll < 0.05:
    fish_type = "Legendary"      # 5%
elif fish_roll < 0.20:
    fish_type = "Rare"          # 15%
elif fish_roll < 0.45:
    fish_type = "Good"          # 25%
else:
    fish_type = "Common"        # 55% 普通鱼

# 特殊鱼种：独立roll，与普通鱼互斥
special_roll = random(0.0, 1.0)
if special_roll < 0.05:
    fish_type = "Bomb"          # 5% 炸弹鱼
elif special_roll < 0.10:
    fish_type = "Lucky"         # 5% 幸运鱼

# 注意：特殊鱼种不与普通/优质/精品/传说叠加
# 即：不会出现"幸运普通鱼"或"炸弹传说鱼"
```

### 2. 积分计算

```
# 单次钓鱼积分
base_score = FISH_BASE_SCORE[fish_type]
# Common=10, Good=25, Rare=50, Legendary=100

rarity_multiplier = FISH_RARITY_MULTIPLIER[fish_type]
# Common=1.0, Good=2.5, Rare=5.0, Legendary=10.0

# 幸运加成：仅对普通/优质/精品/传说鱼类生效
# 炸弹鱼不享受幸运加成
if fish_type != "Bomb":
    lucky_multiplier = 2.0 if is_lucky else 1.0
else:
    lucky_multiplier = 1.0  # 炸弹鱼不受幸运影响

# 速度加成：仅对普通/优质/精品/传说鱼类生效
# 炸弹鱼不享受速度加成
if fish_type != "Bomb" and catch_time < FIGHT_TIME_LIMIT:
    if catch_time < FAST_THRESHOLD:
        speed_multiplier = 1.2      # 快速
    elif catch_time <= SLOW_THRESHOLD:
        speed_multiplier = 1.0      # 正常
    else:
        speed_multiplier = 0.8      # 慢速
else:
    speed_multiplier = 1.0          # 炸弹鱼或超时：固定1.0

final_score = base_score * rarity_multiplier * lucky_multiplier * speed_multiplier

# 炸弹鱼特殊处理：固定-50分
if fish_type == "Bomb":
    final_score = BOMB_PENALTY  # -50
```

> **搏鱼时间来源**：`catch_time` 来自 M01 的 `fishing_complete` 结果中的 `fight_time` 字段，
> 表示搏鱼阶段耗时。M01 GDD 规定搏鱼最长 60 秒（超时则鱼逃跑）。

### 3. 积分下限

```
# 确保不会变成负分
current_score = clamp(current_score + final_score, 0, INF)
```

### 4. 预计比赛结果

```
# 平均每次钓鱼时间
avg_fish_time = 10  # 秒

# 3分钟可钓鱼次数
max_fishes = floor(CONTEST_DURATION / avg_fish_time)
# = floor(180 / 10) = 18 条

# 期望积分 (简化)
expected_score = max_fishes * (0.45 * 10 + 0.25 * 25 + 0.15 * 50 + 0.05 * 100)
# = 18 * 26.75 = 481.5

# 高手积分 (90%成功率)
expert_score = expected_score * 1.5
# = 722
```

### 5. 排名计算

```
# 排名规则: 总积分 > 鱼类数量 > 传说鱼数量
player_rank = 1
for other in players:
    if other.total_score > player.total_score:
        player_rank += 1
    elif other.total_score == player.total_score:
        if other.fish_count > player.fish_count:
            player_rank += 1
        elif other.fish_count == player.fish_count:
            if other.legendary_count > player.legendary_count:
                player_rank += 1
```

### 6. 奖励计算

```
# 名次奖励
rank_rewards = {
    1: {"coins": 300, "title": "钓鱼大赛冠军"},
    2: {"coins": 200, "title": null},
    3: {"coins": 100, "title": null},
    default: {"coins": 50, "title": null}
}

total_coins = rank_rewards[rank].coins

# 额外奖励
if total_score > 500: total_coins += 50
if total_score > 1000: total_coins += 100
if is_new_record: total_coins += 100
```

## Edge Cases

### 1. 积分边界

| 情况 | 处理方式 |
|------|----------|
| **积分变成负数** | 最低为0，不会扣成负分 |
| **连续炸弹鱼** | 每次-50，直到达到0 |
| **幸运鱼连续** | 正常计算，不限制 |
| **幸运鱼+炸弹鱼同时触发** | 不可能发生（独立roll，互斥） |
| **炸弹鱼+速度加成** | 炸弹鱼固定-50分，不享受速度加成 |
| **炸弹鱼+幸运加成** | 炸弹鱼固定-50分，不享受幸运加成 |

### 2. 时间边界

| 情况 | 处理方式 |
|------|----------|
| **最后一秒提竿** | 允许，只要在时间耗尽前开始 |
| **搏鱼超时** | 鱼类逃跑，显示"时间到！" |
| **准备阶段退出** | 返回活动，无惩罚 |

### 3. 鱼类边界

| 情况 | 处理方式 |
|------|----------|
| **连续传说鱼** | 允许，不限制稀有度 |
| **无鱼咬钩** | 等待超时后重新抛竿 |
| **提竿失败** | 鱼类逃跑，不扣分 |

### 4. 排行榜边界

| 情况 | 处理方式 |
|------|----------|
| **单人比赛** | 正常显示排名，无竞争 |
| **同时结束** | 按完成动画顺序排名 |
| **网络断开** | 保存本地成绩，重连后更新 |

### 5. 奖励边界

| 情况 | 处理方式 |
|------|----------|
| **首次冠军** | 只触发一次成就 |
| **连续刷新记录** | 每次都奖励 |
| **背包已满** | 奖励改为金币补偿 |

### 6. 输入边界

| 情况 | 处理方式 |
|------|----------|
| **比赛期间切后台** | 暂停游戏 |
| **游戏崩溃** | 保存当前成绩，重新进入可继续 |
| **同时按多个键** | 只响应第一个有效输入 |

### 7. M01集成边界

| 情况 | 处理方式 |
|------|----------|
| **M01提竿失败** | 鱼类逃跑，进入下一轮等待 |
| **M01超时** | 鱼类逃跑 |
| **搏鱼过程时间到** | 中断搏鱼，鱼类逃跑 |

### 依赖关系说明

**硬依赖**:
- M05 需要M01的时机小游戏和搏鱼机制
- 鱼类数据来自P02钓鱼系统
- 端午节期间才能参与

**与M01的关系**:
- M05是M01的包装，在比赛模式下运行M01
- M05负责计时、积分、排名
- M01负责具体的钓鱼操作

**活动期间**:
