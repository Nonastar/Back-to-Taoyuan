## Overview

赛龙舟小游戏是端午节节日活动中的团队划船竞赛游戏。玩家操控龙舟船头，与3个AI龙舟队伍在河道上进行竞速比赛。游戏采用鼓点节奏玩法，玩家需在鼓点落下时按下划桨按钮，根据节奏准确度和连击次数获得加速加成，最终根据到达终点的名次获得奖励。

设计为 Standard 复杂度，包含：
1. **鼓点节奏系统**：跟随鼓声节奏按键获得分数和加速
2. **划桨机制**：控制划桨节奏，影响船只速度
3. **连击加成**：连续完美按键累积加速效果
4. **AI竞速**：3支AI龙舟队伍，不同难度策略

小游戏独立于主游戏运行，完成后根据名次与活动系统交互进行奖励结算。

## Player Fantasy

赛龙舟小游戏给玩家带来**团队协作的凝聚感和竞技冲刺的紧张感**。玩家应该感受到：

- **鼓声的召唤** — 咚咚咚的鼓点响起，激起内心的节奏感
- **划桨的节奏** — 跟随鼓点一起划动，仿佛置身于龙舟之上
- **连击的爆发** — 连续完美按键带来的加速冲刺，超越对手
- **冲刺的激情** — 最后冲刺阶段，拼命划桨争第一
- **胜利的欢呼** — 第一个冲过终点线的狂喜

参考游戏：
- **节奏天国**: 跟随节奏按键的核心玩法
- **龙舟竞赛游戏**: 传统端午节的龙舟体验
- **划船类体感游戏**: 多人协作的竞技感

**情感曲线**:
1. **起跑阶段**: 准备就绪，鼓声响起，蓄势待发
2. **中段蓄力**: 保持节奏，积累连击，等待爆发时机
3. **连击爆发**: 连续完美按键，速度飙升，超越对手
4. **最后冲刺**: 全力划桨，争夺第一
5. **终点冲刺**: 极限加速，胜负在此一举

## Detailed Design

### Core Rules

#### 1. 鼓点节奏系统

##### 1.1 鼓点生成

```
鼓点间隔: 固定 0.8 秒 (75 BPM)
鼓点提示: 提前 0.3 秒显示"鼓点指示器"

鼓点序列:
  [咚] ── 0.8s ── [咚] ── 0.8s ── [咚] ...

每 4 拍一个小节，每小节结束时有一次"大力划"
```

##### 1.2 按键判定

| 判定 | 时间窗口 | 分数 | 加速效果 |
|------|----------|------|----------|
| **完美 (Perfect)** | ±50ms | 100分 | +5% 速度 |
| **良好 (Good)** | ±100ms | 50分 | +2% 速度 |
| **一般 (OK)** | ±200ms | 20分 | +1% 速度 |
| **错过 (Miss)** | >200ms 或未按 | 0分 | 0% 速度 |
| **大力划 (Power)** | 每小节最后一拍 | 150分 | +10% 速度 |

##### 1.3 节奏指示器

```
屏幕中心显示节奏指示器:

         [时机窗口]
  ←────────────●─────────────→
  左侧区域    中心点    右侧区域

  按键时指示器位置决定判定:
  - 在中心点 ±50ms: 完美
  - 在中心点 ±100ms: 良好
  - 在中心点 ±200ms: 一般
  - 超出窗口: 错过
```

#### 2. 连击加成系统

##### 2.1 连击计数

```
连续完美/良好按键 = 连击数 +1
一般按键 = 连击数 +1 (也计入连击)
Miss = 连击数重置为 0

连击里程碑:
  5 连击: 获得"小爆发"效果
  10 连击: 获得"大爆发"效果
  20 连击: 获得"龙之怒"效果
```

> **注意**: 一般(OK)按键也计入连击数，但不获得速度加成。

##### 2.2 连击效果

| 连击等级 | 触发条件 | 效果 |
|----------|----------|------|
| 无 | 0 连击 | 正常速度 |
| 小爆发 | 5 连击 | 持续 3 秒 +20% 速度 |
| 大爆发 | 10 连击 | 持续 5 秒 +40% 速度 |
| 龙之怒 | 20 连击 | 持续 8 秒 +80% 速度 |

##### 2.3 爆发叠加

```
多个连击效果可以叠加:
  小爆发 + 大爆发 = +60% 速度 (取较长持续时间)
  大爆发 + 龙之怒 = +120% 速度
```

#### 3. 船只移动系统

##### 3.1 基础速度

```
基础速度: 100 单位/秒
玩家速度 = 基础速度 × (1 + 加速加成)

加速加成来源:
  - 节奏按键: +1% ~ +10% (根据判定)
  - 连击效果: +20% ~ +80% (根据等级)
  - 疲劳惩罚: -10% (最后冲刺阶段)
```

##### 3.2 距离计算

```
每帧移动距离 = 当前速度 × delta_time

总距离 = Σ 每帧移动距离
完赛时间 = 总距离 / 平均速度
```

##### 3.3 疲劳系统 (最后冲刺)

```
比赛进行到 80% 距离时进入最后冲刺阶段

最后冲刺阶段:
  - 鼓点速度加快 25% (0.8s → 0.6s)
  - 玩家获得 10% 疲劳惩罚
  - 爆发效果更易触发
```

#### 4. 比赛系统

##### 4.1 参赛队伍

| 队伍 | 描述 | 难度 | AI策略 |
|------|------|------|--------|
| **玩家** | 玩家控制的龙舟 | - | 跟随玩家节奏 |
| **青龙队** | AI队伍 | 简单 | 70% 准确率，保守加速 |
| **红龙队** | AI队伍 | 普通 | 80% 准确率，平衡策略 |
| **金龙队** | AI队伍 | 困难 | 85% 准确率，激进加速 |

##### 4.2 赛道设置

```
赛道长度: 1000 单位
起点: 0 单位
终点: 1000 单位
检查点: 250, 500, 750 单位 (显示进度)
```

##### 4.3 胜负判定

```
名次计算: 按到达终点顺序
  第1名: 金牌
  第2名: 银牌
  第3名: 铜牌
  第4名: 无牌

并列处理: 同时到达终点按距离小数位决定
```

#### 5. 奖励结算

##### 5.1 名次奖励

| 名次 | 金币奖励 | 粽子道具 |
|------|----------|----------|
| 第1名 (金牌) | 500g | 3个 |
| 第2名 (银牌) | 300g | 2个 |
| 第3名 (铜牌) | 150g | 1个 |
| 第4名 | 50g | 0个 |

##### 5.2 额外奖励

| 条件 | 额外奖励 |
|------|----------|
| 首次获胜 | 成就解锁 |
| 完美连击 >10 | +100g |
| 总分 >10000 | +50g |
| 刷新个人记录 | +200g |

## Formulas

### 1. 按键判定时间

```
# 鼓点到达时间
beat_time = beat_index × BEAT_INTERVAL
BEAT_INTERVAL = 0.8  # 秒

# 按键判定
press_time = 玩家按下时间
time_diff = abs(press_time - beat_time)

判定结果 = {
    if time_diff <= 0.050: "Perfect"
    elif time_diff <= 0.100: "Good"
    elif time_diff <= 0.200: "OK"
    else: "Miss"
}
```

### 2. 速度计算

```
# 基础速度
BASE_SPEED = 100  # 单位/秒

# 判定速度加成
hit_bonus = {
    "Perfect": 0.05,   # +5%
    "Good": 0.02,      # +2%
    "OK": 0.01,        # +1%
    "Miss": 0.0        # 无加成
}

# 连击速度加成
combo_bonus = {
    0: 0.0,            # 无
    5: 0.20,           # 小爆发 +20%
    10: 0.40,          # 大爆发 +40%
    20: 0.80           # 龙之怒 +80%
}

# 疲劳惩罚 (最后冲刺阶段)
fatigue_penalty = is_final_sprint ? 0.10 : 0.0

# 最终速度
total_bonus = hit_bonus + combo_bonus - fatigue_penalty
current_speed = BASE_SPEED × (1 + total_bonus)
```

### 3. 距离计算

```
# 每帧距离
frame_distance = current_speed × delta

# 累积距离
total_distance += frame_distance

# 到达终点判定
if total_distance >= TRACK_LENGTH:
    finish_position = current_position
```

### 4. AI速度计算

```
# AI准确率
ai_accuracy = {
    "青龙队": 0.75,   # 简单
    "红龙队": 0.85,   # 普通
    "金龙队": 0.95    # 困难
}

# AI随机判定
is_hit = random() < ai_accuracy

if is_hit:
    hit_result = random() < 0.5 ? "Perfect" : "Good"
    ai_speed = BASE_SPEED × (1 + hit_bonus[hit_result])
else:
    ai_speed = BASE_SPEED
```

### 5. 连击计算

```
# 连续命中计数
combo = previous_combo + 1  # 完美/良好/一般
combo = 0                     # Miss

# 爆发等级
burst_level = {
    if combo >= 20: "DragonRage"
    elif combo >= 10: "BigBurst"
    elif combo >= 5: "SmallBurst"
    else: "None"
}
```

### 6. 名次计算

```
# 到达顺序
finish_order = sorted(boats, key=lambda b: b.finish_time)
player_rank = finish_order.index(player) + 1

# 奖励计算
rank_rewards = {
    1: {"coins": 500, "zongzi": 3},
    2: {"coins": 300, "zongzi": 2},
    3: {"coins": 150, "zongzi": 1},
    4: {"coins": 50, "zongzi": 0}
}

total_reward = rank_rewards[player_rank]
if total_perfect_combo > 10: total_reward.coins += 100
if total_score > 10000: total_reward.coins += 50
if is_new_record: total_reward.coins += 200
```

### 7. 疲劳系统

```
# 进入最后冲刺
if distance / TRACK_LENGTH >= 0.8 and not is_final_sprint:
    is_final_sprint = true
    beat_interval = 0.6  # 加快 25%
    fatigue_penalty = 0.10
```

### 8. 变量定义

| 变量 | 类型 | 范围 | 说明 |
|------|------|------|------|
| `BEAT_INTERVAL` | float | 0.6-1.0s | 鼓点间隔 |
| `BASE_SPEED` | float | 80-120 | 基础速度 |
| `TRACK_LENGTH` | float | 800-1200 | 赛道长度 |
| `PERFECT_WINDOW` | float | 50ms | 完美判定窗口 |
| `GOOD_WINDOW` | float | 100ms | 良好判定窗口 |
| `OK_WINDOW` | float | 200ms | 一般判定窗口 |

| `total_distance` | float | 0-TRACK_LENGTH | 玩家距离 |
| `is_final_sprint` | bool | - | 是否最后冲刺 |

## Edge Cases

### 1. 节奏边界

| 情况 | 处理方式 |
|------|----------|
| **按得太早** | 如果在鼓点前200ms内按，算作错过 |
| **按得太晚** | 如果在鼓点后200ms后按，不计入本次判定 |
| **连续按多次** | 只计算最近一次在窗口内的按键 |
| **鼓点间隔内未按** | 判定为 Miss |

### 2. 连击边界

| 情况 | 处理方式 |
|------|----------|
| **Miss后立即Perfect** | 连击从0开始计，爆发效果立即取消 |
| **爆发效果重叠** | 叠加加速效果 |
| **爆发效果到期** | 平滑过渡回正常速度 |
| **Miss时爆发效果存在** | 爆发效果立即取消 |
| **连击数极高** (>50) | 保持龙之怒效果 |

### 3. 比赛边界

| 情况 | 处理方式 |
|------|----------|
| **所有AI同时到达** | 按随机顺序排名 |
| **玩家和AI同时到达** | 玩家优先 |
| **玩家提前完成** | 等待其他队伍完成 |
| **网络断开/崩溃** | 视为中途退出，记录当前成绩 |

### 4. 疲劳边界

| 情况 | 处理方式 |
|------|----------|
| **爆发效果持续到冲刺结束** | 爆发效果优先于疲劳惩罚 |
| **冲刺阶段获得新爆发** | 重新计时，持续到冲刺结束 |

### 5. 奖励边界

| 情况 | 处理方式 |
|------|----------|
| **背包已满** | 提示"背包已满，奖励已发送至邮箱" |
| **金币溢出** | 使用int64存储 |
| **首次获胜** | 检查本地记录后解锁成就 |
| **刷新记录** | 对比历史最高分 |

### 6. 输入边界

| 情况 | 处理方式 |
|------|----------|
| **触摸和键盘同时输入** | 只计算第一个输入 |
| **按住键不放** | 只计算第一次按下 |
| **快速连点** | 只计算有效窗口内的按键 |
| **游戏暂停时按键** | 忽略所有输入 |

### 7. 性能边界

| 情况 | 处理方式 |
|------|----------|
| **帧率低于30fps** | 降低动画质量，保持判定精度 |
| **帧时间波动大** | 使用平滑delta时间 |

### 8. 音效/动画边界

| 情况 | 处理方式 |
|------|----------|
| **鼓点音效延迟** | 使用预加载，延迟播放 |
| **动画卡顿** | 简化龙舟划桨动画 |
| **BGM与鼓点不同步** | 以鼓点为准，优先保证节奏感 |
