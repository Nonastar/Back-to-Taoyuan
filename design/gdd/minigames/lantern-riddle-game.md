# 猜灯谜小游戏 (Lantern Riddle Game)

> **状态**: Designed
> **Author**: Claude Code
> **Last Updated**: 2026-04-07
> **System ID**: M06
> **Implements Pillar**: Mini-game 玩法
> **Festival**: 元宵节 (农历正月十五)
> **Question Types**: 选择题 + 文字输入

## Overview

猜灯谜小游戏是元宵节节日活动中的文字问答玩法。玩家在限定时间内回答与传统文化相关的谜语，根据答对数量和答题速度获得积分。

游戏采用混合模式：
- **选择题**：简单谜语提供4个选项，适合休闲玩家快速上手
- **文字输入**：困难谜语需要直接输入谜底，考验玩家的文化知识

每场游戏包含10道题目，答对得分、答错不扣分但结束本题。根据总积分发放奖励。

小游戏独立于主游戏运行，完成后根据成绩与活动系统交互进行奖励结算。

## Player Fantasy

猜灯谜小游戏给玩家带来**解谜的成就感和元宵节的温馨氛围**。玩家应该感受到：

- **灯笼的温暖** — 看到精美灯笼亮起，感受到节日氛围
- **解谜的乐趣** — 思考谜面，理解线索，得出答案的乐趣
- **限时的紧张** — 倒计时一秒秒流逝，必须快速作答
- **答对的满足** — "答对了！"的提示带来成就感
- **连对的狂喜** — 连续答对多题后的自信和喜悦
- **学习的收获** — 在游戏中了解传统文化知识

参考游戏：
- **传统猜谜游戏**: 文字解谜的乐趣
- **知识问答游戏**: 限时答题的紧张感
- **元宵节活动**: 赏灯、猜谜的传统节庆体验

**情感曲线**:
1. **开场**: 灯笼亮起，营造元宵节氛围
2. **前期**: 简单谜语热身，建立信心
3. **中期**: 难度逐渐上升，感受文化挑战
4. **后期**: 连续答对，成就感爆发
5. **结算**: 查看成绩和排名，分享乐趣

## Detailed Design

### Core Rules

#### 1. 题库系统

##### 1.1 题目分类

| 分类 | 描述 | 示例 |
|------|------|------|
| **字谜** | 字形、字义、字音相关 | "山上还有山" → "出" |
| **成语** | 四字成语典故 | "守株待兔" 相关谜语 |
| **历史** | 历史人物、事件、典故 | "刘备借荆州" → 谜底 |
| **诗词** | 诗句、诗人、诗意 | "飞流直下三千尺" → 谜底 |
| **物品** | 日用品、自然物、食物 | "红公鸡，绿尾巴" → 谜底 |

##### 1.2 难度分级

| 难度 | 等级 | 基础分 | 题目类型 | 答对得分范围 |
|------|------|--------|----------|--------------|
| **简单** | 1-3 | 80分 | 选择题 | 80-130分 |
| **中等** | 4-6 | 120分 | 选择题/文字输入 | 120-170分 |
| **困难** | 7-10 | 150分 | 文字输入 | 150-200分 |

> **注**：答对得分 = 基础分 + 速度加成(0-50分)

##### 1.3 题目格式

**选择题格式**:
```
谜面: "千条线，万条线，落在水里看不见。(打一自然现象)"
选项: A. 雨  B. 雪  C. 雾  D. 雷
```

**文字输入格式**:
```
谜面: "上有加一笔，打一字"
输入框: [____]
提示: 共4个字符
```

#### 2. 游戏流程

##### 2.1 整体流程

```
┌─────────────────────────────────────────┐
│  1. 准备阶段 (5秒)                      │
│     └─ 显示规则，选择难度                │
├─────────────────────────────────────────┤
│  2. 答题阶段 (共10题)                   │
│     └─ 每题30秒，选择或输入答案          │
│           ↓                             │
│     [显示谜面]                          │
│           ↓                             │
│     [选择/输入答案]                     │
│           ↓                             │
│     [判定对错]                          │
│           ↓                             │
│     [显示结果]                          │
│           ↓                             │
│     [下一题或结算]                      │
├─────────────────────────────────────────┤
│  3. 结算阶段                            │
│     └─ 显示成绩 → 发放奖励              │
└─────────────────────────────────────────┘
```

##### 2.2 每题流程

```
显示谜面 → 开始计时(30秒) → 玩家作答 → 判定
    ↓                                     ↓
超时自动跳过                      正确/错误提示
    ↓                                     ↓
下一题                          显示正确答案
```

#### 3. 积分系统

##### 3.1 基础积分

```
答对得分 = 基础分 + 速度加成

基础分（根据难度）:
  简单题: 80分
  中等题: 120分
  困难题: 150分

速度加成:
  剩余时间比例 = 剩余秒数 / 30
  速度加成 = floor(剩余时间比例 × 50)
```

##### 3.2 最高分计算

```
简单题最高分: 80 + 50 = 130分
中等题最高分: 120 + 50 = 170分
困难题最高分: 150 + 50 = 200分

每场最高分 (3简单+4中等+3困难):
  = 3×130 + 4×170 + 3×200
  = 390 + 680 + 600 = 1670分
```

##### 3.4 答错处理

```
答错: 不扣分，显示正确答案，本题结束
超时: 不扣分，显示正确答案，自动下一题
```

#### 4. 奖励结算

##### 4.1 分数段奖励

| 分数段 | 称号 | 金币奖励 |
|--------|------|----------|
| ≥1600分 | 灯谜大师 | 500g |
| ≥1300分 | 猜谜高手 | 300g |
| ≥900分 | 灯谜新秀 | 150g |
| ≥500分 | 参与奖 | 50g |

> **注**：最高分 1670 分，灯谜大师需要达到 96% 正确率。调整为：≥1600分门槛

##### 4.2 额外奖励

| 条件 | 额外奖励 |
|------|----------|
| 首次获得"灯谜大师" | 成就解锁 |
| 连续答对10题 | +200g |
| 刷新个人最高分 | +100g |

#### 5. 每日限制

```
每日参与次数: 3次
每日重置: 每日凌晨6点
超出限制: 提示"今日次数已用完"
```

#### 详细状态定义

| 状态 | 描述 | 入口条件 | 出口条件 | 持续时间 |
|------|------|----------|----------|----------|
| `Idle` | 未开始 | 系统初始化 | 点击开始 | - |
| `Ready` | 准备阶段 | 开始游戏 | 倒计时结束 | 5s |
| `Playing` | 答题进行 | 准备完毕 | 10题完成或退出 | 约5分钟 |
| `Result` | 显示结果 | 游戏结束 | 玩家确认 | 3-5s |

```
┌─────────────┐
│  SHOWING   │ ← 显示谜面
└──────┬──────┘
       │ 显示完成
       ↓
┌─────────────┐
│  ANSWERING │ ← 等待作答
└──────┬──────┘
       │ 玩家作答/超时
       ↓
┌─────────────┐
│  FEEDBACK   │ ← 显示答案结果
└──────┬──────┘
       │ 显示完成
       ↓
┌─────────────┐
│  NEXT       │ ← 下一题
└──────┬──────┘
       │ 有下一题/全部完成
       ↓
┌─────────────┐
│  SHOWING   │ 或  GAME_RESULT
└─────────────┘
```

#### 状态转换表

| 当前状态 | 事件 | 下一状态 | 触发条件 |
|----------|------|----------|----------|
| Idle | `start_game()` | Ready | 玩家点击开始 |
| Ready | 倒计时结束 | Playing | 准备完毕 |
| Playing/SHOWING | 显示完成 | ANSWERING | 谜面显示 |
| Playing/ANSWERING | 玩家提交答案 | FEEDBACK | 作答判定 |
| Playing/ANSWERING | 超时30秒 | FEEDBACK | 时间耗尽 |
| Playing/FEEDBACK | 显示完成 | NEXT | 显示1秒后 |
| Playing/NEXT | 有下一题 | SHOWING | 继续答题 |
| Playing/NEXT | 无下一题 | Result | 全部完成 |
| Playing | 全部答错 | Result | 10题均错误 |
| Result | 玩家确认 | Idle | 返回活动 |

## Formulas

### 1. 积分计算

```
# 单题得分
base_score = BASE_SCORE[difficulty_level]
# 简单(1-3): 80, 中等(4-6): 120, 困难(7-10): 150

time_remaining_ratio = remaining_time / QUESTION_TIME_LIMIT
# QUESTION_TIME_LIMIT = 30秒

speed_bonus = floor(time_remaining_ratio × SPEED_BONUS_MAX)
# SPEED_BONUS_MAX = 50

final_score = base_score + speed_bonus
```

### 2. 速度加成表

| 剩余时间 | 时间比例 | 速度加成 |
|----------|----------|----------|
| 30秒 | 100% | +50 |
| 25秒 | 83% | +41 |
| 20秒 | 67% | +33 |
| 15秒 | 50% | +25 |
| 10秒 | 33% | +16 |
| 5秒 | 17% | +8 |
| 1秒 | 3% | +1 |
| 0秒 | 0% | +0 |

### 3. 难度等级映射

```
get_difficulty_level(difficulty: int) -> String:
    if difficulty <= 3:
        return "easy"      # 简单
    elif difficulty <= 6:
        return "medium"    # 中等
    else:
        return "hard"      # 困难

get_base_score(difficulty_level: String) -> int:
    if difficulty_level == "easy":
        return 80
    elif difficulty_level == "medium":
        return 120
    else:  # "hard"
        return 150
```

## Edge Cases

### 1. 答题边界

| 情况 | 处理方式 |
|------|----------|
| **超时未作答** | 显示正确答案，不扣分，自动下一题 |
| **答错** | 显示正确答案，不扣分，本题结束 |
| **连续答错10题** | 游戏结束，显示最终成绩 |
| **提前交卷** | 可以主动提交，进入结算 |
| **不作答直接退出** | 不保存成绩，不消耗次数 |

### 2. 输入边界

| 情况 | 处理方式 |
|------|----------|
| **输入为空** | 提示"请输入答案"，不提交 |
| **输入过长** | 限制最多20个字符 |
| **特殊字符** | 过滤非中文/英文/数字字符 |
| **大小写** | 答案比较前统一转为小写 |
| **多余空格** | 答案比较前去除空格 |
| **多答案变体** | 支持标准答案的多个写法 |

### 3. 时间边界

| 情况 | 处理方式 |
|------|----------|
| **最后一秒作答** | 允许，只要在超时时提交 |
| **最后一秒切换题目** | 不允许，必须先作答 |
| **倒计时10秒警告** | 播放警告音效，时间变红 |
| **暂停游戏** | 暂停计时，暂停BGM |
| **游戏崩溃恢复** | 恢复后从当前题目继续（如果支持存档）|

### 4. 题目边界

| 情况 | 处理方式 |
|------|----------|
| **题库不足** | 按现有题目重复抽取，标记已答过的题 |
| **题目重复** | 同一题在同一场次不重复出现 |
| **题型切换** | 简单题→选择题，中等题→混合，困难题→文字输入 |
| **题目分类显示** | 显示谜面、选项(如有)、字数提示(文字输入) |

### 5. 积分边界

| 情况 | 处理方式 |
|------|----------|
| **积分超过上限** | 最高分为理论上限1670分 |
| **积分为0** | 正常结算，无称号，50金币参与奖 |
| **分数段临界** | ≥1600得"灯谜大师"，严格按≥判断 |
| **历史最高分更新** | 对比本地存储的最高分 |

### 6. 奖励边界

| 情况 | 处理方式 |
|------|----------|
| **背包已满** | 奖励改为金币补偿 |
| **首次灯谜大师** | 只触发一次成就 |
| **连续刷新记录** | 每次都奖励 |
| **金币溢出** | 使用int64存储 |
| **金币为负** | 不可能发生，奖励只增不减 |

### 7. 每日限制边界

| 情况 | 处理方式 |
|------|----------|
| **今日次数已用完** | 按钮置灰，显示"今日次数已用完" |
| **跨日重置** | 每日凌晨6点重置 |
| **日期切换时参与** | 按进入时的日期计算次数 |
| **每日次数独立** | 每人独立计数，不共享 |

### 8. 选择题边界

| 情况 | 处理方式 |
|------|----------|
| **随机选项生成** | 正确选项随机位置，其他3个为同难度干扰项 |
| **干扰项来源** | 同一类别/难度的其他题目答案 |
| **选项重复** | 4个选项不重复 |
| **触碰选项** | 触屏点击即可选择，高亮选中状态 |

### 9. 文字输入边界

| 情况 | 处理方式 |
|------|----------|
| **显示字数提示** | 显示"谜底X个字" |
| **输入长度限制** | 与正确答案长度一致时自动提交(可选) |
| **支持拼音输入** | 是，允许全拼或简拼 |
| **支持手写输入** | 可选，触屏设备额外支持 |

### 10. 网络/设备边界

| 情况 | 处理方式 |
|------|----------|
| **网络断开** | 本地计时，成绩本地保存 |
| **重连后上传** | 联网后自动上传成绩 |
| **低端设备** | 简化动画，保证核心功能 |
| **横竖屏切换** | 保持当前题目，重新布局UI |

### 11. 题库内容边界

| 情况 | 处理方式 |
|------|----------|
| **题目过于冷门** | 审核时过滤，确保常识性题目 |
| **答案争议** | 支持多个正确变体 |
| **生僻字** | 避免使用，确保玩家能输入 |
| **文化差异** | 选择全球华人通用的谜语 |
