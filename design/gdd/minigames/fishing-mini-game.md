# 钓鱼小游戏 (Fishing Mini-Game)

> **状态**: Designed
> **Author**: Claude Code
> **Last Updated**: 2026-04-07
> **System ID**: M01
> **Implements Pillar**: Mini-game 玩法

## Overview

钓鱼小游戏是钓鱼系统的核心交互模块。玩家抛竿后，浮标在水面浮动等待鱼咬钩；当鱼咬钩时，浮标下沉并显示"最佳时机"区域，玩家需在此时按下提竿按钮；提竿成功后进入"搏鱼"阶段，玩家通过控制保持鱼在绿色区域中，待鱼的力量条耗尽后成功捕获。

设计为 Standard 复杂度，包含两个核心机制：
1. **时机小游戏**：观察浮标状态，在最佳时机提竿
2. **搏鱼小游戏**：保持鱼在绿色区域，消耗鱼的力量条

小游戏状态独立于主游戏（时间暂停），完成后根据结果与 P02 钓鱼系统交互。

## Player Fantasy

钓鱼小游戏给玩家带来**宁静中的期待与紧张感**。玩家应该感受到：

- **等待的宁静** — 浮标轻轻晃动，背景音乐舒缓
- **咬钩的兴奋** — 浮标突然下沉，心跳加速
- **提竿的紧张** — 盯着"最佳时机"区域，屏息等待
- **搏鱼的刺激** — 鱼左右冲刺，手心出汗
- **成功的满足** — 看到鱼跃出水面的成就感

参考游戏：
- **Stardew Valley**: 简单但上瘾，紧张感来自时机把控
- **动物森友会**: 宁静祥和的钓鱼体验

## Detailed Design

### Core Rules

#### 1. 时机小游戏（Timing Mini-Game）

时机小游戏是钓鱼的核心操作环节。玩家需要在正确时机提竿才能进入搏鱼阶段。

##### 1.1 浮标状态

浮标有四种状态，通过视觉和动画区分：

| 状态 | 触发条件 | 视觉表现 | 时机判定 |
|------|----------|----------|----------|
| `Floating` | 抛竿后 | 轻微上下浮动 | 无效（鱼未咬钩）|
| `Bobbing` | 鱼在试探 | 轻微下沉 10-30% | 太早（吓跑鱼）|
| `Sinking` | 鱼咬钩 | 快速下沉 70-100% | **最佳时机** |
| `Diving` | 鱼咬钩后 | 浮标完全消失 | 太晚（鱼逃跑）|

##### 1.2 最佳时机窗口

- **窗口触发**: 浮标下沉超过 70%
- **窗口持续**: `1.0s + (鱼类难度 × 0.1s)`，最长 2.0s
- **浮标下沉速度**: 2.5秒从0到100%（完全消失）
- **视觉提示**:
  - 浮标变红并闪烁
  - 显示"提竿！"文字
  - 音效：心跳加速

##### 1.3 提竿判定

```
提竿结果 = {
    if 当前状态 == "Sinking" 且在最佳时机窗口内:
        return "完美时机"      # 成功率 +15%
    elif 当前状态 == "Sinking" 且时机窗口已过但浮标未消失:
        return "正常时机"     # 进入搏鱼阶段（有轻微惩罚）
    elif 当前状态 == "Bobbing":
        return "太早"         # 鱼被吓跑，无搏鱼
    elif 当前状态 == "Diving":
        return "太晚"         # 鱼逃脱，无搏鱼
    else:
        return "无效"         # 未咬钩就提竿，无搏鱼
}
```

> **实现说明**: 时机窗口结束后、浮标完全消失前（约0.75秒的缓冲时间），玩家提竿仍可进入搏鱼阶段，但判定为"正常时机"而非"完美时机"，给予玩家一定容错空间。

#### 2. 搏鱼小游戏（Fish Fighting Mini-Game）

搏鱼小游戏在提竿成功后触发。玩家需要通过控制保持鱼在安全区域。

##### 2.1 游戏界面

搏鱼界面为横向进度条：

```
┌────────────────────────────────────────────────────────┐
│  ←─────────────[安全区]─────────────→                    │
│  红       黄         绿         黄        红           │
│  0%      20%        50%        80%       100%         │
│                          ●                            │
│                    (鱼的指示器)                        │
│                                                        │
│  压力: ████████████░░░░░░░░░░░░  55%                  │
│  力量: ██████░░░░░░░░░░░░░░░░░░░  25%                  │
└────────────────────────────────────────────────────────┘
```

**界面元素**:
- **鱼的指示器**: 白色圆点，表示鱼当前位置
- **安全区域**: 中间绿色区域（30%-70%），保持在此可消耗鱼的力量
- **危险区域**: 两侧红色区域（0-20% 或 80-100%），在此会快速增加压力
- **过渡区域**: 黄色区域（20-30% 或 70-80%），轻微增加压力
- **压力条**: 玩家操作产生的压力，满则失败
- **力量条**: 鱼的剩余体力，耗尽则成功

##### 2.2 鱼的冲刺行为

根据鱼类难度（1-10），鱼有不同的冲刺模式：

| 难度等级 | 冲刺速度 | 冲刺间隔 | 冲刺幅度 |
|----------|----------|----------|----------|
| 1-3 (简单) | 慢 (1.5s) | 长 (3s) | 小 (30%) |
| 4-6 (中等) | 中 (1.0s) | 中 (2s) | 中 (50%) |
| 7-10 (困难) | 快 (0.5s) | 短 (1s) | 大 (80%) |

**鱼的冲刺逻辑**:
- 鱼会**向逃离方向冲刺**（70%概率）或随机方向（30%概率）
  - 逃离方向定义：鱼在中心左侧(<0.5)时逃离方向为左(-1)，在右侧(>0.5)时逃离方向为右(+1)
- 冲刺幅度基于难度计算
- 冲刺速度基于难度：简单鱼慢，困难鱼快
- 冲刺完成后，鱼会短暂休息再下一次冲刺
- **无输入时**：鱼会自然趋向危险区（自然拉力）

##### 2.3 玩家操作

| 按键 | 动作 | 效果 |
|------|------|------|
| `←` / `A` | 向左收线 | 指示器向左移动 |
| `→` / `D` | 向右收线 | 指示器向右移动 |
| 无输入 | 放线 | 指示器趋向危险区（鱼的自然拉力）|

**移动速度计算**:
```
玩家移动速度 = 基础速度 × 技能加成 × 鱼竿加成

基础速度 = 50% 宽度/秒
技能加成 = 1.0 + (钓鱼技能等级 × 0.05)
鱼竿加成 = 竹竿(1.0) / 玻璃钢(1.1) / 钛合金(1.2) / 传说(1.3)
```

##### 2.4 区域判定与数值变化

每帧（1/60s）更新：

```
if 指示器位置 in 安全区域(30-70%):
    压力变化 = -0.002           # 缓慢降低
    力量消耗 = 0.05             # 消耗鱼的力量

elif 指示器位置 in 过渡区域(20-30% 或 70-80%):
    压力变化 = +0.005           # 缓慢增加
    力量消耗 = 0.03             # 中等消耗

elif 指示器位置 in 危险区域(0-20% 或 80-100%):
    压力变化 = +0.015           # 快速增加
    力量消耗 = 0.02             # 少量消耗
```

#### 3. 成功与失败条件

##### 3.1 成功条件

- **鱼的力量条 ≤ 0**: 鱼力竭，成功捕获
- 触发 `fishing_success` 信号

##### 3.2 失败条件

| 条件 | 原因 |
|------|------|
| 压力条 ≥ 1.0 | 压力过大，鱼挣脱逃跑 |
| 时间超过 60s | 鱼逃脱（超时）|
| 提竿太早/太晚 | 直接失败，无搏鱼阶段 |

##### 3.3 结果输出

搏鱼结束后，输出结果给 P02 钓鱼系统：

```gdscript
# 结果数据结构
结果 = {
    "success": bool,              # 是否成功捕获
    "fish_id": String,            # 鱼类 ID
    "quality": String,            # "normal" | "fine" | "excellent"
    "timing_result": String,      # "perfect" | "normal" | "too_early" | "too_late"
    "fight_time": float          # 搏鱼耗时（秒）
}

# 信号定义
signal fishing_complete(result: Dictionary)
```

#### 4. 玩家操作流程

```
1. [抛竿] 玩家在钓鱼点点击抛竿
       ↓
2. [等待] 浮标在水面浮动，显示等待状态
       ↓
3. [咬钩] 随机时间后（2-10秒），浮标下沉
       ↓
4. [提竿] 玩家观察浮标，在最佳时机按下提竿键
       ↓
   ├─ 完美时机 → 进入搏鱼阶段，成功率+15%
   ├─ 太早 → 鱼被吓跑，钓鱼失败
   └─ 太晚 → 鱼逃跑，钓鱼失败
       ↓
5. [搏鱼] 玩家控制 ← → 保持鱼在安全区域
       ↓
   ├─ 力量耗尽 → 捕获成功
   └─ 压力满/超时 → 鱼逃跑，钓鱼失败
       ↓
6. [结算] 显示捕获结果，存入背包
```

| 手柄 | `A` 按钮 | 左摇杆 或 `LB` / `RB` |
| 触屏 | 提竿按钮 | 左滑/右滑手势 |

#### 6. 辅助模式（Accessibility）

| 功能 | 效果 | 触发条件 |
|------|------|----------|
| 放大安全区 | 安全区扩大至 40-60% | 辅助模式开启 |
| 简化时机 | 最佳时机窗口延长 50% | 辅助模式开启 |
| 显示时机提示 | 实时显示当前时机质量 | 辅助模式开启 |
| 减速鱼冲刺 | 冲刺速度降低 30% | 辅助模式开启 |

### States and Transitions

#### 小游戏状态机

```
┌─────────┐
│  IDLE   │ ← 钓鱼小游戏未激活
└────┬────┘
     │ 开始钓鱼 (P02 调用 start_minigame)
     ↓
┌─────────┐
│ WAITING │ ← 浮标浮动，等待鱼咬钩
└────┬────┘
     │ 鱼咬钩 (随机 2-10 秒)
     ↓
┌─────────┐
│  BITE   │ ← 浮标下沉，进入时机判定
└────┬────┘
     │ 玩家提竿
     ↓
┌─────────┐      ┌─────────┐
│ REELING │ ←──→ │ FISHING │ ← 搏鱼阶段
└────┬────┘      └────┬────┘
     │                │
     │ (提竿失败)     │ (成功/失败)
     ↓                ↓
┌─────────┐      ┌─────────┐
│ FAILED  │      │ SUCCESS │ 或  FAILED
└────┬────┘      └────┬────┘
     │                │
     └───────┬────────┘
             │ 完成结算
             ↓
         [返回 P02]
```

#### 详细状态定义

| 状态 | 描述 | 入口条件 | 出口条件 | 持续时间 |
|------|------|----------|----------|----------|
| `Idle` | 钓鱼小游戏未激活 | 系统初始化 | P02 调用 start | - |
| `Waiting` | 浮标浮动等待 | start_minigame | 鱼咬钩 | 2-10 秒 |
| `Bite` | 浮标下沉，时机判定 | 鱼咬钩事件 | 玩家提竿 | 0.3-0.8s |
| `Reeling` | 提竿中 | 时机判定成功 | 进入搏鱼 | 瞬间 |
| `Fishing` | 搏鱼进行中 | Reeling 完成 | 成功/失败 | 1-60s |
| `Success` | 捕获成功 | 力量条耗尽 | 结算完成 | 2s |
| `Failed` | 捕获失败 | 压力满/超时/时机失败 | 结算完成 | 2s |

## Formulas

### 1. 时机小游戏公式

#### 1.1 咬钩时间

```
基础咬钩时间 = random(2.0, 10.0) 秒

鱼饵缩短系数:
  普通饵料: 0.9
  美味饵料: 0.8
  传说饵料: 0.6

技能缩短系数 = 1.0 - (钓鱼技能等级 × 0.02)

最终咬钩时间 = 基础咬钩时间 × 鱼饵缩短系数 × 技能缩短系数
```

#### 1.2 最佳时机窗口

```
基础窗口时间 = 1.0 秒

鱼类难度加成 = fish.difficulty × 0.1 秒
(难度 1-10 → 加成 0.1-1.0 秒)

最终窗口时间 = min(基础窗口时间 + 鱼类难度加成, 2.0 秒)
```

| 难度 | 窗口时间 | 说明 |
|------|----------|------|
| 1-3 | 1.1s - 1.3s | 简单鱼，窗口较短但可玩 |
| 4-6 | 1.4s - 1.6s | 中等鱼，需要一定反应 |
| 7-10 | 1.7s - 2.0s | 困难鱼，窗口较长以保持可玩性 |

> **实现说明**: 浮标下沉速度为2.5秒完全消失，时机窗口触发点为下沉70%。玩家在时机窗口结束后、浮标完全消失前提竿，仍可进入搏鱼阶段（判定为"正常时机"），给予容错空间。

#### 1.3 提竿成功率

```
基础成功率 = 50%

时机加成:
  完美时机: +15%
  正常时机: +0%
  太早: -30%
  太晚: -50%

最终成功率 = clamp(基础成功率 + 时机加成, 5%, 95%)
```

### 2. 搏鱼小游戏公式

#### 2.1 鱼冲刺速度

```
基础冲刺时间 = 根据难度等级:
  简单 (1-3): 1.5 秒
  中等 (4-6): 1.0 秒
  困难 (7-10): 0.5 秒

冲刺间隔 = 基础冲刺时间 × (1.0 + random(0.5, 1.5))

冲刺目标位置:
  简单: ±15-30% (中心为 50%)
  中等: ±25-50%
  困难: ±40-80%
```

#### 2.2 玩家收线速度

```
基础速度 = 50% 宽度/秒

技能加成系数 = 1.0 + (钓鱼技能等级 × 0.05)

鱼竿加成系数:
  竹竿: 1.0
  玻璃钢竿: 1.1
  钛合金竿: 1.2
  传说鱼竿: 1.3

最终收线速度 = 基础速度 × 技能加成系数 × 鱼竿加成系数
```

#### 2.3 力量消耗

```
安全区域消耗 = 0.05/帧
过渡区域消耗 = 0.03/帧
危险区域消耗 = 0.02/帧

每帧 = 1/60 秒

安全区域每秒消耗 = 0.05 × 60 = 3.0
过渡区域每秒消耗 = 0.03 × 60 = 1.8
危险区域每秒消耗 = 0.02 × 60 = 1.2
```

#### 2.4 压力增长

```
安全区域压力变化 = -0.002/帧 (每秒 -0.12)
过渡区域压力变化 = +0.005/帧 (每秒 +0.30)
危险区域压力变化 = +0.015/帧 (每秒 +0.90)
```

#### 2.5 搏鱼时长估算

```
鱼的初始力量 = 100%

每秒总消耗 ≈ 3.0 (在安全区) - 0.12 (压力减少) = 2.88

理想时长 = 100% / 2.88 ≈ 35 秒

如果玩家一直在危险区:
  每秒消耗 ≈ 1.2 - 0.90 = 0.30
  最长时长 ≈ 333 秒 (超过 60 秒超时限制)
```

### 3. 品质加成公式

```
基础高品质概率 = 10%

技能加成 = 钓鱼技能等级 × 2%

时机加成:
  完美时机: +5%
  正常时机: +0%

最终高品质概率 = clamp(基础高品质概率 + 技能加成 + 时机加成, 0%, 30%)
```

## Edge Cases

### 1. 时机小游戏边界情况

| 情况 | 处理方式 |
|------|----------|
| 提竿太早（鱼未咬钩）| 显示"鱼被吓跑了"，钓鱼失败，无体力消耗 |
| 提竿太早（Bobbing 状态）| 显示"太早了！"，钓鱼失败，无体力消耗 |
| 提竿太晚（浮标消失）| 显示"太晚了，鱼跑了"，钓鱼失败，无体力消耗 |
| 连续 3 次太早提竿 | 后续咬钩概率降低 20%（持续 5 分钟）|
| 完美时机窗口内多次按压 | 只计算第一次提竿 |
| 窗口刚结束时按压 | 判定为"太晚" |

### 2. 搏鱼小游戏边界情况

| 情况 | 处理方式 |
|------|----------|
| 压力条瞬间满（极端操作）| 立即失败，触发脱钩动画 |
| 指示器在边界位置 | 按边界值计算区域 |
| 指示器在安全区边缘 | 按实际位置计算，不四舍五入 |
| 玩家不操作 | 指示器自动趋向危险区（鱼的自然拉力）|
| 按键过快（连续按压）| 每帧只响应一次输入，无效按键被忽略 |
| 玩家同时按左右 | 最近一次按键优先 |

### 3. 特殊情况

| 情况 | 处理方式 |
|------|----------|
| 搏鱼超时（60s）| 显示"鱼逃跑了"，钓鱼失败 |
| 体力不足（搏鱼中）| 继续搏鱼，结束后体力扣除 |
| 存档/加载（搏鱼中）| 不允许，钓鱼小游戏期间禁止存档 |
| 游戏最小化（搏鱼中）| 暂停小游戏 |
| 网络断开（联机模式）| 暂停并显示等待连接 |

> **设计建议**：考虑添加最小捕获时间（如 5s），确保即使玩家操作正确也需要一定时间，增加游戏节奏感。

### 4. 物品/装备边界

| 情况 | 处理方式 |
|------|----------|
| 没有鱼饵 | 竹竿可无饵钓鱼，咬钩时间+50% |
| 装备传说鱼竿 | 所有加成取最大值 |
| 技能等级 0 | 使用基础数值，无加成 |
| 背包满 | 捕获成功但显示"背包已满，存入临时背包" |

### 5. 传说鱼特殊处理

| 情况 | 处理方式 |
|------|----------|
| 传说鱼脱钩 | 当日该传说鱼不再出现 |
| 传说鱼搏鱼失败 | 统计为"脱钩"，不计入捕获数 |
| 传说鱼搏鱼成功 | 触发特殊成功动画和音效 |

### 6. 辅助模式边界

| 情况 | 处理方式 |
|------|----------|
| 辅助模式 + 高难度鱼 | 窗口时间 = 基础 × 1.5，仍有挑战性 |
| 辅助模式 + 传说鱼 | 窗口时间最多延长至 1.2s |
| 关闭辅助模式 | 立即使用标准参数 |

### 7. 输入设备切换

| 情况 | 处理方式 |
|------|----------|
| 游戏中切换输入设备 | 立即生效，无需暂停 |
| 手柄断开连接 | 自动切换到键盘/触屏 |
| 触屏设备切换到键鼠 | 隐藏触屏按钮，显示键盘提示 |

### 8. 性能边界

| 情况 | 处理方式 |
|------|----------|
| 帧率低于 30fps | 小游戏时间同步减速，保持公平 |
| 帧率低于 15fps | 显示性能警告，建议降低画质 |
| 帧时间波动大 | 使用平滑的时间增量计算 |

### 9. 并发情况

| 情况 | 处理方式 |
|------|----------|
| 多个钓鱼点同时触发 | 只能激活一个小游戏 |
| 小游戏进行中再次抛竿 | 无效，忽略操作 |
| 退出钓鱼点时小游戏进行中 | 自动取消，返回 P02 |

### UI 元素规格

| 元素 | 规格 | 交互 |
|------|------|------|
| **浮标** | 32x32px sprite，下沉动画 | 视觉状态指示 |
| **时机提示文字** | 红色，"提竿！"，32px 字体，闪烁动画 | 0.3-0.8s 显示 |
| **鱼的指示器** | 16x16px 白色圆点 | 跟随位置移动 |
| **安全/危险区** | 半透明色块，宽度 40%/20%/20% | 无交互 |
| **力量条** | 200x20px，蓝色渐变，从右向左减少 | 显示当前值 |
| **压力条** | 200x20px，红色渐变，从左向右增加 | 显示当前值 |
| **提竿按钮** | 80x40px (触屏)，键盘显示"空格" | 按下触发提竿 |
| **收线提示** | 左右箭头指示器 | 显示可用的收线方向 |
