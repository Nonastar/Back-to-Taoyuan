> **状态**: In Design

> **Festival**: 中秋节 (Mid-Autumn Festival)

## Overview

品茶会小游戏是中秋节节日活动中的休闲品鉴玩法。玩家通过观察茶汤颜色、闻茶香、品味茶韵，鉴别不同品种的茶叶。

游戏采用品鉴判断模式：
- **观色**：观察茶汤颜色深浅
- **闻香**：辨别茶香类型和浓度
- **品味**：感受茶汤的口感层次

每场游戏有5轮品鉴机会，根据判断正确率获得积分。

小游戏独立于主游戏运行，完成后根据成绩与活动系统交互进行奖励结算。

## Player Fantasy

品茶会小游戏给玩家带来**雅致悠闲的品茗体验和知识渊博的成就感**。玩家应该感受到：

- **茶香的韵味** — 闻着淡淡的茶香，感受宁静致远的心境
- **色泽的奥妙** — 观察茶汤从翠绿到琥珀的变化
- **口感的层次** — 品味苦、涩、甘、甜的微妙平衡
- **鉴别的乐趣** — 挑战自己的味觉和嗅觉敏感度
- **文化的底蕴** — 感受中国传统茶文化的魅力

参考游戏：
- **茶道物语**: 茶道仪式的仪式感
- **品酒大师**: 味觉鉴别系统的专业感
- **星露谷物语**: 休闲轻松的节日氛围

**情感曲线**:
1. **开场**: 中秋庭院，茶香袅袅，宁静氛围
2. **前期**: 熟悉品鉴流程，感受茶韵
3. **中期**: 尝试鉴别不同茶叶，掌握技巧
4. **后期**: 挑战高难度茶叶，追求完美
5. **品茗**: 欣赏品鉴结果，分享心得

**与中秋节的关联**:
- 中秋节赏月品茗是传统习俗
- 茶香与桂花香交织的秋夜
- 团圆赏月时的悠闲时光

## Detailed Design

### Core Rules

#### 1. 游戏设置

```
品鉴轮次: 5轮
每轮品鉴: 观色 → 闻香 → 品味 → 选择答案
游戏时长: 每轮约10秒思考时间
茶品种类: 6种基础茶叶 + 2种特殊茶
```

#### 2. 品鉴三要素

##### 2.1 观色（视觉判断）

```
观察茶汤颜色，从以下维度判断：
- 茶汤色泽：翠绿/嫩绿/黄绿/金黄/琥珀/红褐
- 清澈度：清澈/微浊/浑浊
- 亮度：明亮/暗淡

提示：不同茶叶有独特的色泽特征
```

##### 2.2 闻香（嗅觉判断）

```
辨别茶香类型：
- 嫩香：清新草叶香
- 花香：茉莉/桂花香
- 果香：蜜桃/苹果香
- 栗香：炒制坚果香
- 陈香：老茶特有的醇香

浓度等级：无/淡/中/浓
```

##### 2.3 品味（味觉判断）

```
品味口感层次：
- 苦味：微弱/适中/强烈
- 涩味：平滑/收敛/刺激
- 甘甜：清甜/回甘/蜜甜
- 醇厚：清淡/爽口/醇厚

茶韵：生津/回甘/喉韵
```

#### 3. 茶叶类型

##### 3.1 基础茶叶

| 茶叶 | 色泽 | 香气 | 口感特点 | 难度 |
|------|------|------|----------|------|
| **龙井** | 黄绿 | 嫩香/豆香 | 清鲜甘醇 | ★★☆ |
| **碧螺春** | 翠绿 | 花香果香 | 鲜醇回甘 | ★★☆ |
| **铁观音** | 金黄 | 兰花香 | 醇厚甘滑 | ★★★ |
| **大红袍** | 琥珀 | 岩骨花香 | 醇厚甘爽 | ★★★ |
| **普洱** | 红褐 | 陈香枣香 | 醇滑甜润 | ★★☆ |
| **茉莉花茶** | 黄绿 | 茉莉花香 | 鲜醇甘爽 | ★☆☆ |

##### 3.2 特殊茶叶（高难度）

| 茶叶 | 色泽 | 香气 | 口感特点 | 难度 |
|------|------|------|----------|------|
| **白毫银针** | 浅黄 | 毫香清鲜 | 清甜醇和 | ★★★★ |

#### 4. 品鉴流程

```
每轮品鉴流程：
1. 展示茶汤（3秒）
2. 观察颜色提示
3. 点击"闻香"按钮（2秒闻香动画）
4. 香气类型提示
5. 点击"品味"按钮（2秒品味动画）
6. 口感描述提示
7. 选择茶叶（4秒）
8. 显示结果
```

#### 5. 判定系统

##### 5.1 正确判定

```
完全正确：观色✓ + 闻香✓ + 品味✓ = 满分
部分正确：答对1-2项 = 部分分
完全错误：0分
```

##### 5.2 判定示例

```
茶汤提示：色泽金黄、香气浓郁、口感醇厚
正确答案：大红袍

判定：
- 选择大红袍 = 完全正确，300分
- 选择铁观音 = 部分正确（色泽接近），150分
- 选择龙井 = 完全错误，0分
```

## Formulas

### 1. 茶叶匹配度计算

```
匹配度 = (颜色匹配项 + 香气匹配项 + 口感匹配项) / 3 × 100%
```

**判定规则**:

```gdscript
func calculate_match(tea_a: Dictionary, tea_b: Dictionary) -> float:
    color_match = tea_a.color == tea_b.color ? 1.0 : 0.0
    aroma_match = tea_a.aroma_type == tea_b.aroma_type ? 1.0 : 0.5  # 部分匹配
    taste_match = tea_a.taste_profile == tea_b.taste_profile ? 1.0 : 0.5
    return (color_match + aroma_match + taste_match) / 3.0
```

### 2. 单轮得分计算

```
单轮得分 = 基础分 × 匹配度系数
```

**变量定义**:

| 变量 | 含义 | 范围 |
|------|------|------|
| `BASE_SCORE` | 基础分 | 100~300 |
| `match_coefficient` | 匹配度系数 | 0.0 ~ 1.0 |

**匹配度系数映射**:

| 匹配度 | 判定 | 得分系数 |
|--------|------|----------|
| 100% | 完全正确 | 1.0 |
| 66.7% | 大部分正确 | 0.5 |
| 33.3% | 小部分正确 | 0.25 |
| 0% | 完全错误 | 0.0 |

### 3. 基础分计算

```
基础分 = 茶叶难度分 × 时间奖励
```

**茶叶难度分**:

| 难度星级 | 基础分 |
|----------|--------|
| ★☆☆ | 100分 |
| ★★☆ | 150分 |
| ★★★ | 200分 |
| ★★★★ | 300分 |

**时间奖励**:

```
if 回答时间 <= 2秒: 时间奖励 = 1.2
elif 回答时间 <= 3秒: 时间奖励 = 1.1
elif 回答时间 <= 4秒: 时间奖励 = 1.0
else: 时间奖励 = 0.8
```

### 4. 连击加成

```
连击奖励 = COMBO_BONUS[当前连击数]
```

**连击加成表**:

| 连击数 | 额外加分 |
|--------|----------|
| 2连 | +20分 |
| 3连 | +40分 |
| 4连 | +80分 |
| 5连 | +150分 |

**连击判定**:

```
if 匹配度 >= 66.7%:
    combo_count += 1
    combo_bonus = COMBO_BONUS[combo_count]
else:
    combo_count = 0
    combo_bonus = 0
```

### 5. 最终得分计算

```
最终得分 = Σ(单轮得分) + Σ(连击奖励) + 额外奖励
```

**额外奖励**:

| 条件 | 额外奖励 |
|------|----------|
| 5轮全对 | +200g |
| 刷新个人最高分 | +100g |
| 首次获得"品茶大师" | 成就解锁 |

## Edge Cases

### 1. 游戏状态异常

#### 1.1 品鉴过程中游戏中断

```
场景: 玩家在品鉴过程中强制退出游戏
处理:
  - 保存当前品鉴进度
  - 如果正在品鉴的茶叶已显示，该轮计0分
  - 重新进入时从当前进度继续
```

#### 1.2 5轮全部超时

```
场景: 玩家每轮都超时未选择
处理:
  - 每轮计0分
  - 显示结果，称号判定: <300分 → 茶童
```

#### 1.3 同时满足多个称号条件

```
场景: 分数同时满足多个段位
处理:
  - 取最高称号（如1200分同时满足"品茶大师"和"茶艺师"）
  - 只发放最高段位奖励
```

### 2. 品鉴判定异常

#### 2.1 两个茶叶匹配度相同

```
场景: 选择时有两个茶叶匹配度相同
处理:
  - 计分按匹配度最高的茶叶计
  - 如果多个最高匹配度相同，计第一个
```

#### 2.2 提示揭示时机

```
场景: 玩家在闻香阶段直接跳过
处理:
  - 允许跳过，但香气提示不显示
  - 视为放弃该维度信息
```

#### 2.3 答案选择限制

```
场景: 玩家犹豫不决导致超时
处理:
  - 超时自动选择最高匹配度的茶叶
  - 按超时计分（时间奖励 = 0.8）
```

### 3. 连击系统异常

#### 3.1 第一轮失败后第二轮成功

```
场景: 第一轮匹配度<66.7%，第二轮匹配度100%
处理:
  - 连击从第二轮重新计算
  - 第二轮计为连击起点（combo=1）
```

#### 3.2 最后一轮与连击判定

```
场景: 第五轮（最后一轮）后连击达到5连
处理:
  - 第五轮计入连击
  - 显示"5连达成！"提示
  - 连击奖励加入总分
  - 游戏直接进入Result
```

### 4. 时间相关异常

#### 4.1 每日重置恰好在游戏中

```
场景: 每日凌晨6点重置时玩家正在进行游戏
处理:
  - 当前游戏局不计入当日次数
  - 完成后显示"今日次数已用完"
```

### 5. 网络/存档异常

#### 5.1 奖励结算时网络断开

```
场景: 游戏完成但奖励结算时网络断开
处理:
  - 分数和称号本地保存
  - 奖励标记为"待领取"
  - 重新连接后补发奖励
```

#### 5.2 本地存档损坏

```
场景: 最高分记录存档损坏
处理:
  - 检测到损坏时重置为0
  - 不影响当前游戏
```

### 6. 茶叶选择异常

#### 6.1 茶叶重复出现

```
场景: 同一茶叶在多轮中出现
处理:
  - 允许重复
  - 每轮独立计分
  - 不影响游戏公平性
```

#### 6.2 选项茶叶相似度过高

```
场景: 两个茶叶的色、香、味非常接近
处理:
  - 提供额外区分提示
  - 提示玩家注意细微差别
```

### 7. 用户操作异常

#### 7.1 快速跳过所有介绍

```
场景: 玩家跳过所有介绍直接开始
处理:
  - 允许跳过
  - 保持完整品鉴流程
```

#### 7.2 重复点击品鉴按钮

```
场景: 玩家快速重复点击确认按钮
处理:
  - 忽略重复点击
  - 等待动画播放完毕
```

### 8. 难度平衡异常

#### 8.1 5轮全对过于困难

```
场景: 5轮全对获得率接近0
处理:
  - 降低部分茶叶的区分难度
  - 增加提示信息的清晰度
```

#### 8.2 5轮全对过于简单

```
场景: 5轮全对获得率超过50%
处理:
  - 提高茶叶区分难度
  - 减少可用提示
```
