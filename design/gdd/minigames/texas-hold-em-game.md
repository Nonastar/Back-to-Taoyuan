## Overview

德州扑克小游戏是沙漠赌场系统的核心赌博游戏。玩家坐在牌桌前，与 3 个 AI 对手进行标准德州扑克对决。游戏包含翻牌前、翻牌、转牌、河牌四个下注轮次，以及最终的摊牌判定。小游戏接收 P14 赌场系统的赌注配置和金币结算回调，实现完整的扑克游戏逻辑。

设计为 Standard 复杂度，包含：
1. **发牌系统**：洗牌、发底牌、发公共牌
2. **下注系统**：跟注、加注、弃牌、全下
3. **AI 决策**：基于手牌强度和位置的智能决策
4. **胜负判定**：标准德州扑克牌型比较

小游戏独立于主游戏运行，完成后根据结果与 P14 交互进行金币结算。

## Player Fantasy

德州扑克小游戏给玩家带来**策略博弈的智识感和翻牌时的紧张刺激**。玩家应该感受到：

- **发牌的期待** — 底牌翻开的那一刻，屏息凝神看自己的牌
- **翻牌的惊喜** — 三张翻牌揭开时，心跳加速，期待形成好牌型
- **下注的抉择** — 加注还是弃牌？这个决定让玩家感觉自己掌控全局
- **摊牌的胜负** — 亮牌时刻，有人欢呼有人叹息
- **与 AI 的心理战** — 观察 AI 的下注模式，尝试读懂它们的牌

参考游戏：
- **Red Dead Redemption 2 的扑克**: 沉浸式的赌场氛围和真实的扑克体验
- **Stardew Valley 的赌场**: 简单但上瘾的赌博机制
- **任何在线扑克游戏**: 策略与运气的完美结合

**情感曲线**:
1. **拿到好牌**: 兴奋、期待、思考如何最大化收益
2. **拿到烂牌**: 谨慎、纠结、考虑弃牌还是 bluff
3. **翻牌形成听牌**: 希望、紧张、计算胜率
4. **摊牌胜利**: 满足、成就感
5. **摊牌失败**: 懊恼、但想再来一局

## Detailed Design

### Core Rules

#### 1. 扑克牌系统

##### 1.1 牌组定义

```
一副扑克牌 = 52 张
花色: 黑桃(♠)、红心(♥)、梅花(♣)、方块(♦)
点数: A(14)、K(13)、Q(12)、J(11)、10、9、8、7、6、5、4、3、2

Ace 可作高牌(A-K-Q-J-10)也可作低牌(A-2-3-4-5)
```

##### 1.2 牌型大小（从大到小）

| 牌型 | 说明 | 示例 |
|------|------|------|
| **皇家同花顺** | 同花色的 A-K-Q-J-10 | A♥K♥Q♥J♥10♥ |
| **同花顺** | 同花色的连续5张 | 9♥8♥7♥6♥5♥ |
| **四条** | 四张相同点数的牌 | K♠K♥K♣K♦9♠ |
| **葫芦** | 三条 + 一对 | Q♠Q♥Q♣3♦3♠ |
| **同花** | 同花色的任意5张 | A♠J♠8♠6♠2♠ |
| **顺子** | 连续5张（花色不同）| 10♦9♣8♥7♠6♣ |
| **三条** | 三张相同点数的牌 | 7♠7♥7♣K♦4♠ |
| **两对** | 两个对子 | 9♠9♥5♣5♦J♦ |
| **一对** | 两张相同点数的牌 | A♠A♥K♦Q♣J♠ |
| **高牌** | 无上述牌型，取最大单张 | A♠K♦Q♣J♥9♠ |

**牌型比较规则**:
```
1. 先比较牌型大小（皇家同花顺 > 同花顺 > ... > 高牌）
2. 同牌型比较点数：四条 > 三条 > 葫芦 > ... > 一对 > 高牌
3. 同点数比较单张：依次比较最大单张、次大单张...
4. 完全相同时为平局，底池平分
```

#### 2. 游戏流程

##### 2.1 牌桌配置

```
每桌玩家数: 4人（1个玩家 + 3个AI）
玩家位置: 随机分配（开局时决定）

位置顺序（顺时针）:
  [Button] → [SB] → [BB] → [UTG] → [MP] → [CO] → [Button]
  Btn=庄家位, SB=小盲, BB=大盲, UTG=枪口位, MP=中位, CO=关位
```

##### 2.2 游戏阶段

```
┌─────────────────────────────────────────────────────────┐
│  1. 发牌阶段 (Dealing)                                  │
│     └─ 洗牌 → 发底牌(每人2张)                           │
├─────────────────────────────────────────────────────────┤
│  2. 翻牌前下注 (Pre-flop)                              │
│     └─ 大盲/小盲下注 → 顺时针行动 → 达到均衡            │
├─────────────────────────────────────────────────────────┤
│  3. 翻牌下注 (Flop)                                    │
│     └─ 发3张公共牌 → 下注轮                             │
├─────────────────────────────────────────────────────────┤
│  4. 转牌下注 (Turn)                                    │
│     └─ 发1张公共牌 → 下注轮                             │
├─────────────────────────────────────────────────────────┤
│  5. 河牌下注 (River)                                   │
│     └─ 发1张公共牌 → 下注轮                             │
├─────────────────────────────────────────────────────────┤
│  6. 摊牌 (Showdown)                                    │
│     └─ 比较牌型 → 分配底池 → 结束                        │
└─────────────────────────────────────────────────────────┘
```

##### 2.3 盲注系统

```
小盲 (SB): 位置在庄家左边的玩家，下注最低赌注的一半
大盲 (BB): 位置在小盲左边的玩家，下注最低赌注

例如低档赌注(10-50g):
  SB = 5g
  BB = 10g

翻牌前行动从 SB 左边的玩家开始
```

#### 3. 玩家行动

##### 3.1 可用行动

| 行动 | 说明 | 条件 |
|------|------|------|
| **弃牌 (Fold)** | 放弃本局，失去已下注筹码 | 任何时候 |
| **跟注 (Call)** | 下注与当前最高下注相等 | 下注 > 0 |
| **过牌 (Check)** | 不下注，通过 | 当前最高下注 = 0 |
| **加注 (Raise)** | 提高下注额 | 下注 ≥ 最低赌注 |
| **全下 (All-in)** | 下注所有剩余筹码 | 筹码 > 0 |

##### 3.2 下注限制

```
最低加注额 = 当前最高下注 + (当前最高下注 - 上一轮最高下注)
  或 = 最低赌注（取较大值）

例如:
  当前下注: 50g
  上一轮下注: 20g
  最低加注额 = 50 + (50 - 20) = 80g
```

##### 3.3 全下 (All-in) 规则

```
玩家全下时:
1. 只能赢得与其全下金额相等的底池部分（边池）
2. 其他玩家可以继续竞争主池
3. 全下后自动进入摊牌（如所有玩家都已行动）

示例:
  玩家A: 100g全下
  玩家B: 跟注100g
  玩家C: 弃牌
  底池: 200g，全部归A或B
```

#### 4. AI 决策系统

##### 4.1 AI 行为参数（来自 P14 配置）

```
AI 根据大厅等级有不同的激进度:

标准厅: aggression = 0.3 (保守)
中级厅: aggression = 0.5 (中等)
高级厅: aggression = 0.8 (激进)
```

##### 4.2 AI 手牌评估

AI 手牌评估使用所有 7 张牌（2 张底牌 + 5 张公共牌）组成最佳 5 张牌来计算强度。

```gdscript
# 计算手牌强度 (0.0 - 1.0)
func evaluate_hand_strength(pocket: Array[Card], community: Array[Card]) -> float:
    # 组成最佳5张牌（从7张中选择）
    best_hand = get_best_5_cards(pocket + community)

    # 基础牌型分数
    hand_type_score = {
        "high_card": 0.0,
        "pair": 1.0,
        "two_pair": 2.0,
        "three_of_kind": 3.0,
        "straight": 4.0,
        "flush": 5.0,
        "full_house": 6.0,
        "four_of_kind": 7.0,
        "straight_flush": 8.0,
        "royal_flush": 9.0
    }

    base_score = hand_type_score[best_hand.type]

    # 点数加成（同牌型时，高点数更强）
    kicker_score = 0.0
    for i, card in enumerate(best_hand.kickers):
        kicker_score += card.rank / 14.0 * pow(0.1, i)

    # 最终强度 (0.0 - 1.0)
    # 皇家同花顺 = 1.0
    strength = (base_score + kicker_score) / 9.0

    return clamp(strength, 0.0, 1.0)
```

> **注**: AI 手牌评估基于完整 7 张牌的最佳组合，而非仅底牌。

##### 4.3 AI 决策逻辑

```
每轮下注时，AI 执行以下决策:

1. 计算手牌强度 (0.0 - 1.0)
2. 根据激进度调整:
   - 高激进度: 更容易加注、更愿意 bluff
   - 低激进度: 更保守、只在好牌时加注

3. 决策阈值:
   if 手牌强度 < 0.2:
       if 当前下注 > 0: 弃牌
       else: 过牌
   elif 手牌强度 < 0.5:
       if 当前下注 < 最低赌注 * 2: 跟注/过牌
       else: 弃牌
   elif 手牌强度 < 0.8:
       if 可加注且 rand() < 激进度: 加注
       else: 跟注
   else:
       加注/全下
```

##### 4.4 AI 位置调整

位置参数映射（顺时针从庄家位开始）:

| position 值 | 位置 | 描述 | 强度调整 |
|------------|------|------|----------|
| 0 | Button (Btn) | 庄家位 | -0.02（位置优势最大）|
| 1 | Small Blind (SB) | 小盲 | +0.05 |
| 2 | Big Blind (BB) | 大盲 | +0.05 |
| 3 | Under the Gun (UTG) | 枪口位 | +0.08（位置劣势最大）|

> **注**: AI 在靠前位置需要更强的手牌才能行动，靠后位置可以更激进。

#### 5. 胜负结算

##### 5.1 底池分配

```
单池情况:
  胜者获得全部底池

多池情况 (有玩家全下):
  主池: 所有玩家都可争夺
  边池: 全下玩家只能赢得与其全下金额相等的部分

示例:
  玩家A: 50g全下
  玩家B: 跟注50g后全下100g
  玩家C: 跟注150g
  底池: 300g

  主池: 50g * 4 = 200g (A/B/C 争夺)
  边池1: 50g * 3 = 150g (B/C 争夺)
  边池2: 50g * 2 = 100g (C独有)
```

##### 5.2 结果回调 P14

```gdscript
# 游戏结束时，向 P14 发送结果
结果 = {
    "success": bool,              # 玩家是否获胜
    "pot_amount": int,            # 底池总额
    "player_win": int,            # 玩家赢得金额
    "player_loss": int,           # 玩家损失金额
    "hand_result": String,        # 玩家的最佳牌型
    "winning_hand": String,        # 获胜者的牌型
    "game_duration": float        # 游戏时长（秒）
}

emit_signal("poker_hand_ended", 结果)
```

```gdscript
    # 组成最佳5张牌
    best_hand = get_best_5_cards(pocket + community)

    # 基础牌型分数
    hand_type_score = {

        "pair": 1.0,
        "two_pair": 2.0,
        "three_of_kind": 3.0,
        "straight": 4.0,
        "flush": 5.0,
        "full_house": 6.0,
        "four_of_kind": 7.0,
        "straight_flush": 8.0,
        "royal_flush": 9.0
    }

    base_score = hand_type_score[best_hand.type]

    # 点数加成（同牌型时，高点数更强）
    kicker_score = 0.0
    for i, card in enumerate(best_hand.kickers):
        kicker_score += card.rank / 14.0 * pow(0.1, i)

    # 最终强度 (0.0 - 1.0)
    # 皇家同花顺 = 1.0
    strength = (base_score + kicker_score) / 9.0

    return clamp(strength, 0.0, 1.0)
```

### 2. 同花听牌概率

```
# 转牌/河牌形成同花的概率
outs = 同花剩余张数
turn_probability = outs / 47  # 转牌前，47张未知牌
river_probability = outs / 46  # 转牌后，46张未知牌

# 转牌或河牌形成同花
either_probability = 1 - ((47 - outs) / 47) * ((46 - outs) / 46)

示例:
  手牌: A♥K♥
  转牌前 outs = 9 (剩余红心)
  either_probability ≈ 35%
```

### 3. 顺子听牌概率

```
outs = 顺子剩余张数
# 计算方式同同花

常见听牌 outs:
  两端顺子 (如 9-10): 8张
  卡张顺子 (如 9-J): 4张
  双端顺子: 15张 (含同花听牌)
```

### 4. AI 决策公式

```
# AI 加注概率
def calculate_raise_probability(hand_strength: float, aggression: float, position: int) -> float:
    base_prob = hand_strength * 0.5

    # 激进度加成
    aggression_bonus = aggression * 0.3

    # 位置调整 (后面位置更容易加注)
    position_bonus = position * 0.02  # 每位置 +2%

    raise_prob = base_prob + aggression_bonus + position_bonus

    return clamp(raise_prob, 0.0, 1.0)

# AI 跟注概率
def calculate_call_probability(hand_strength: float, aggression: float, pot_odds: float) -> float:
    # Pot Odds: 跟注额 / (底池 + 跟注额)
    pot_odds_needed = 1.0 - hand_strength

    if pot_odds > pot_odds_needed:
        # 有正期望值，更容易跟注
        call_prob = 0.5 + (pot_odds - pot_odds_needed) * aggression
    else:
        # 需要冒险
        call_prob = hand_strength * aggression

    return clamp(call_prob, 0.0, 1.0)
```

### 5. 底池赔率 (Pot Odds)

```
Pot Odds = 跟注额 / (当前底池 + 跟注额)

示例:
  底池 = 100g
  需要跟注 = 50g
  Pot Odds = 50 / (100 + 50) = 0.33 = 33%

# 与手牌强度比较
if 手牌强度 > Pot Odds:
    跟注有正期望值
else:
    弃牌更优
```

### 6. 诈唬 (Bluff) 概率

```
# AI 诈唬概率
bluff_probability = aggression * (1.0 - hand_strength) * recent_bet_ratio

# recent_bet_ratio: 近期加注频率加成
# 如果 AI 最近频繁加注，诈唬可信度降低
```

### 7. 预期价值 (Expected Value)

```
EV = (胜率 × 赢得金额) - (败率 × 损失金额)

示例:
  跟注 50g
  当前底池 150g
  胜率 = 40%

  EV = (0.4 × 200) - (0.6 × 50) = 80 - 30 = +50g

  EV > 0: 跟注有利
  EV < 0: 弃牌有利
```

## Edge Cases

### 1. 发牌边界

| 情况 | 处理方式 |
|------|----------|
| 扑克牌用完 | 自动重新洗牌 |
| 玩家断线 | AI 继续游戏，玩家自动弃牌 |
| 游戏最小化 | 暂停游戏，等待玩家返回 |

### 2. 下注边界

| 情况 | 处理方式 |
|------|----------|
| 玩家金币 < 最低跟注 | 自动全下或弃牌 |
| 加注超过最高限额 | 自动降低为最高限额 |
| 加注 < 最低加注额 | 自动提高为最低加注额 |
| 玩家金币刚好等于下注额 | 提示全下选项 |

### 3. AI 行为边界

| 情况 | 处理方式 |
|------|----------|
| AI 无足够金币跟注 | AI 自动全下或弃牌 |
| 所有 AI 都弃牌 | 玩家独赢底池 |
| AI 连续获胜过多 | 不干预（风险由玩家承担）|
| AI 决策超时 | 使用保守策略（跟注或弃牌）|

### 4. 摊牌边界

| 情况 | 处理方式 |
|------|----------|
| 多人牌型相同 | 底池平分 |
| 无法平分（奇数金币）| 多出的 1 金币归庄家（系统）|
| 玩家弃牌后其他玩家摊牌 | 不显示其他玩家的牌 |

### 5. 边池边界

| 情况 | 处理方式 |
|------|----------|
| 玩家全下但被超过 | 玩家只能赢得与其全下相等的部分 |
| 多个玩家全下 | 按比例分配边池 |
| 全下玩家弃牌 | 边池由剩余玩家争夺 |

### 6. 特殊情况

| 情况 | 处理方式 |
|------|----------|
| 玩家在第一轮弃牌 | 显示"再来一局"选项 |
| 玩家选择离开牌桌 | 按弃牌处理 |
| 玩家在赌博中退出游戏 | 视为弃牌，损失当前下注 |
| 中途关闭游戏 | 下次进入时恢复，但赌博进度丢失 |

### 7. 性能边界

| 情况 | 处理方式 |
|------|----------|
| 帧率低于 30fps | AI 决策时间延长，保持公平 |
| 手牌计算延迟 | 使用异步计算，不阻塞 UI |

### 8. 公平性边界

| 情况 | 处理方式 |
|------|----------|
| 玩家金币不足一局 | 禁止坐下，显示"金币不足" |
| AI 作弊风险 | AI 只能看到自己的手牌，不访问玩家手牌 |
| 牌组随机性 | 使用加密随机数生成器保证公平 |
