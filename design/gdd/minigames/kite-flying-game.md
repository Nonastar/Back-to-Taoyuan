# 放风筝小游戏 (Kite Flying Game)

> **状态**: In Design
> **Author**: Claude Code
> **Last Updated**: 2026-04-08
> **System ID**: M11
> **Implements Pillar**: Mini-game 玩法
> **Festival**: 清明节 (Qingming Festival)

## Overview

放风筝小游戏是清明节节日活动中的户外休闲玩法。玩家通过控制风筝的升降和方向，在春风中放飞风筝，完成各种任务目标。

游戏采用风向控制模式：
- **风力调节**：控制风筝的升降高度
- **方向控制**：左右移动避开障碍物
- **任务挑战**：完成图案绘制或穿越目标

每场游戏有3次放飞机会，根据完成度和任务表现获得积分。

小游戏独立于主游戏运行，完成后根据成绩与活动系统交互进行奖励结算。

## Player Fantasy

放风筝小游戏给玩家带来**春风拂面的轻快和童年嬉戏的温馨回忆**。玩家应该感受到：

- **春风的律动** — 感受春风的力量和方向变化
- **风筝的轻盈** — 看着风筝在蓝天中摇曳飘舞
- **自由的翱翔** — 操控风筝穿越云层的成就感
- **任务的挑战** — 完成各种图案和穿越任务的乐趣
- **童年的回忆** — 重温清明踏青放风筝的传统

参考游戏：
- **Temple Run**: 无限跑酷的节奏感
- ** Alto's Adventure**: 滑雪的流畅控制
- **风筝模拟器**: 风筝飞行的真实感

**情感曲线**:
1. **开场**: 清明时节，春风和煦，草地绿意
2. **前期**: 熟悉风力控制，感受风筝飘动
3. **中期**: 尝试穿越目标，掌握飞行技巧
4. **后期**: 挑战高难度任务，追求高分
5. **欣赏**: 观看风筝在蓝天中飞舞

**与清明节的关联**:
- 清明时节是放风筝的传统习俗
- 春风吹拂是放风筝的最佳时机
- 踏青郊游时的户外活动

## Detailed Design

### Core Rules

#### 1. 游戏设置

```
放飞次数: 3次
游戏时长: 每次约30秒飞行时间
风力变化: 风向和强度随机变化
任务类型: 穿越目标 / 绘制图案
```

#### 2. 操作控制

##### 2.1 风力控制

```
操作: 按住屏幕左侧控制风筝升降
- 按住: 风筝上升
- 松开: 风筝下降

重力影响:
- 无操作时风筝缓慢下降
- 下降速度随高度增加而加快
```

##### 2.2 方向控制

```
操作: 左右倾斜设备或滑动屏幕
- 左倾/左滑: 风筝左移
- 右倾/右滑: 风筝右移

移动特点:
- 移动速度与风力相关
- 顺风时移动更快
- 逆风时移动更慢
```

#### 3. 任务类型

##### 3.1 穿越目标

```
任务描述: 操控风筝穿越随机出现的圆环
圆环位置: 随机分布在飞行路径上
完成判定: 风筝中心穿过圆环区域

任务目标:
- 连续穿越5个圆环 = 基础分
- 连续穿越10个圆环 = 额外奖励
```

##### 3.2 绘制图案

```
任务描述: 操控风筝沿指定路径飞行
路径显示: 半透明引导线
完成判定: 风筝沿路径飞行80%以上

图案难度:
- 直线: 简单
- 波浪线: 中等
- 螺旋: 困难
```

#### 4. 风力系统

##### 4.1 风力方向

```
风向类型:
- 东风: 风筝向右飘
- 西风: 风筝向左飘
- 南风: 风筝上升加快
- 北风: 风筝下降加快

风向变化: 每10秒随机切换方向
```

##### 4.2 风力强度

```
强度等级: 1~5级
- 1级: 微风，风筝稳定
- 3级: 正常，适中挑战
- 5级: 强风，风筝剧烈摆动

强度变化: 根据高度增加
```

#### 5. 奖励结算

##### 5.1 分数段奖励

| 分数段 | 称号 | 金币奖励 |
|--------|------|----------|
| ≥2000分 | 风筝大师 | 500g |
| ≥1500分 | 风筝高手 | 300g |
| ≥1000分 | 风筝新手 | 150g |
| ≥500分 | 参与者 | 50g |

##### 5.2 额外奖励

| 条件 | 额外奖励 |
|------|----------|
| 单次穿越10连 | +100g |
| 首次获得"风筝大师" | 成就解锁 |
| 刷新个人最高分 | +100g |

#### 6. 每日限制

```
每日参与次数: 3次
每日重置: 每日凌晨6点
超出限制: 提示"今日次数已用完"
```

## Formulas

### 1. 风筝物理模型

```
风筝位置更新 = 位置 + 速度 × dt
风筝速度更新 = 速度 + 加速度 × dt
```

**受力分析**:

```
上升力 = 玩家输入 × UPWARD_FORCE
重力 = -GRAVITY × 风筝质量
风力 = WIND_FORCE × 风力强度 × cos(风筝角度 - 风向)
阻力 = -DRAG × 速度 × |速度|
总加速度 = (上升力 + 重力 + 风力 + 阻力) / 质量
```

**参数定义**:

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `UPWARD_FORCE` | 20 N | 上升推力 |
| `GRAVITY` | 9.8 m/s² | 重力加速度 |
| `WIND_FORCE` | 5 N | 基础风力 |
| `DRAG` | 0.5 | 空气阻力系数 |
| `KITE_MASS` | 1.0 kg | 风筝质量 |
| `HEIGHT_TOLERANCE` | 20px | 穿越高度容差 |

### 2. 穿越判定

```
穿越判定 = is_inside_circle(kite_position, target_position, target_radius)
           AND is_within_height_range(kite_y, target_y, height_tolerance)
           AND is_correct_direction(kite_velocity, target_direction)
```

**穿越条件**:
- 风筝中心在圆环范围内（水平距离 < 目标半径）
- 风筝高度在圆环高度范围内（垂直距离 < 高度容差）
- 飞行方向正确（从上往下穿越，velocity_y > 0）

**判定公式**:

```gdscript
func check_crossing(kite: Kite, target: Target) -> bool:
    # 水平距离检查
    horizontal_dist = abs(kite.x - target.x)
    if horizontal_dist > target.radius:
        return false

    # 高度范围检查
    vertical_dist = abs(kite.y - target.y)
    if vertical_dist > HEIGHT_TOLERANCE:
        return false

    # 飞行方向检查（从上往下）
    if kite.velocity_y <= 0:
        return false

    return true
```

### 3. 连击计算

```
连击奖励分 = TARGET_COMBO_BONUS[当前连击数]
```

**连击加成表**:

| 连击数 | 额外加分 |
|--------|----------|
| 3连 | +30分 |
| 5连 | +80分 |
| 10连 | +200分 |

### 4. 单次飞行得分

```
单次得分 = Σ(穿越得分) + Σ(连击奖励) - 坠落惩罚
```

**穿越得分**:

| 穿越类型 | 得分 |
|----------|------|
| 大圆环 | 50分 |
| 中圆环 | 100分 |
| 小圆环 | 150分 |

**坠落惩罚**:

```
坠落次数 × 100分
```

### 5. 最终得分计算

```
最终得分 = Σ(单次飞行得分) + 额外奖励
```

### 6. 称号判定

```
称号 = evaluate_title(final_score)
```

| 分数段 | 称号 | 金币奖励 |
|--------|------|----------|
| ≥2000分 | 风筝大师 | 500g |
| ≥1500分 | 风筝高手 | 300g |
| ≥1000分 | 风筝新手 | 150g |
| ≥500分 | 参与者 | 50g |
| <500分 | 鼓励奖 | 0g |

### 7. 风力计算

```
effective_wind = base_wind × (1 + height_factor) × intensity_modifier

height_factor = kite_height / MAX_HEIGHT
intensity_modifier = 1.0 + (random() - 0.5) × 0.2
```

### 8. 圆环生成算法

```gdscript
func generate_targets(flight_path: Array) -> Array:
    targets = []
    for i in range(target_count):
        # 随机选择位置
        x = random_range(MARGIN, SCREEN_WIDTH - MARGIN)
        y = random_range(MIN_HEIGHT, MAX_HEIGHT)
        # 随机大小
        size = random_choice([SMALL, MEDIUM, LARGE])
        # 添加目标
        targets.append(Target.new(x, y, size))
    return targets
```

## Edge Cases

### 1. 游戏状态异常

#### 1.1 飞行过程中游戏中断

```
场景: 玩家在飞行过程中强制退出游戏
处理:
  - 保存当前飞行状态
  - 计入当前飞行得分
  - 重新进入时从当前进度继续
```

#### 1.2 3次全部坠落

```
场景: 玩家3次飞行都坠落
处理:
  - 显示结果画面
  - 称号判定: <500分 → 鼓励奖
```

#### 1.3 同时满足多个称号条件

```
场景: 分数同时满足多个段位
处理:
  - 取最高称号
  - 只发放最高段位奖励
```

### 2. 穿越判定异常

#### 2.1 风筝擦边穿过

```
场景: 风筝边缘刚好接触圆环
处理:
  - 仅中心点在范围内才算穿越
  - 边缘接触不计分
```

#### 2.2 多个圆环重叠

```
场景: 两个圆环位置重叠
处理:
  - 同时穿越两个圆环
  - 分别计分
  - 触发双倍连击奖励
```

### 3. 风力系统异常

#### 3.1 风向突变

```
场景: 风向突然变化导致风筝失控
处理:
  - 风向渐变过渡（不是瞬间切换）
  - 防止玩家无法反应
```

#### 3.2 极端风力

```
场景: 5级强风导致风筝剧烈摆动
处理:
  - 限制最大摆动幅度
  - 防止风筝飞出屏幕
```

### 4. 时间相关异常

#### 4.1 每日重置恰好在游戏中

```
场景: 每日凌晨6点重置时玩家正在进行游戏
处理:
  - 当前游戏局不计入当日次数
  - 完成后显示"今日次数已用完"
```

### 5. 网络/存档异常

#### 5.1 奖励结算时网络断开

```
场景: 游戏完成但奖励结算时网络断开
处理:
  - 分数和称号本地保存
  - 奖励标记为"待领取"
  - 重新连接后补发奖励
```

### 6. 控制异常

#### 6.1 设备旋转导致方向混乱

```
场景: 设备从横屏切换到竖屏
处理:
  - 锁定横屏方向
  - 或者重新校准控制
```

#### 6.2 多点触控冲突

```
场景: 同时触摸多个位置
处理:
  - 优先响应第一个触摸点
  - 忽略后续触摸
```

### 7. 性能异常

#### 7.1 低帧率导致物理模拟不稳定

```
场景: 设备性能不足导致帧率波动
处理:
  - 使用固定时间步长物理模拟
  - 不受帧率影响
```

### 7. 调优建议

**降低难度**:
- 增大圆环尺寸
- 减少目标数量
- 降低风力强度
- 减小坠落惩罚

**提高难度**:
- 减小圆环尺寸
- 增加目标数量
- 提高风力强度
- 增大坠落惩罚

**平衡测试重点**:
- 验证风筝大师获得率 < 10%
- 验证平均穿越率在 60%~80%
- 验证每次飞行平均得分在 300~600
