# 隐藏NPC系统 (Hidden NPC System)

> **状态**: Approved
> **Author**: Claude Code
> **Last Updated**: 2026-04-07
> **System ID**: P07
> **Implements Pillar**: 社交系统与NPC互动

## Overview

隐藏NPC系统（仙灵系统）为游戏添加了6位神秘的隐藏角色，玩家通过探索、供奉和互动逐步发现并建立深厚羁绊。每位仙灵都有独特的能力、发现链、缘分等级和结缘奖励。系统与多个生产系统联动，提供独特的被动加成，是游戏深度和长期目标的的重要组成部分。

## Player Fantasy

隐藏NPC系统给玩家带来**神秘感和探索的惊喜**。玩家应该感受到：

- **发现的惊喜** — 意外触发仙灵的线索，发现隐藏的精灵
- **互动的期待** — 每日供奉和独特互动提升缘分
- **能力的惊喜** — 解锁被动能力带来惊喜加成
- **羁绊的深情** — 结缘后的每日奖励和专属内容

**Reference games**: Stardew Valley 的隐藏 Willy 的内容；My Time at Portia 的隐藏角色；原神的命座系统。

| 品质 | 乘数 |
|------|------|
| normal | ×1.0 |
| fine | ×1.25 |

| supreme | ×2.0 |

- 每日供奉1次

#### 6. 求缘与结缘

**求缘门槛**: 1800缘分
**结缘门槛**: 2500缘分

**求缘流程**:
1. 满足求缘门槛
2. 消耗求缘物品
3. 进入求缘状态
4. 继续供奉提升缘分
5. 满足结缘门槛

**结缘流程**:
1. 满足结缘门槛
2. 消耗结缘物品
3. 完成结缘
4. 获得结缘奖励

**结缘规则**:
- 同时只能与一位仙灵求缘/结缘
- 结缘物品昂贵（需制作）
- 可花费10000g解除羁绊

## Formulas

### 1. 供奉缘分公式

```
# 基础缘分
base_affinity = {
    "resonant": 100,    # 灵犀
    "pleased": 50,     # 合意
    "neutral": 10,     # 一般
    "repelled": -40    # 排斥
}

# 品质乘数
quality_multiplier = {
    "normal": 1.0,
    "fine": 1.25,
    "excellent": 1.5,
    "supreme": 2.0
}

# 显灵日乘数
manifestation_multiplier = 3.0

# 最终缘分
final_affinity = base_affinity[category] * quality_multiplier * manifestation_multiplier
```

### 2. 独特互动缘分公式

**参悟（龙灵）**:
```
affinity_gain = (farming_level + foraging_level + fishing_level + mining_level) * 3
```

**奏乐（月兔）**:
```
affinity_gain = 30 + rand(0, 20)  # 30-50
```

**祭仪（桃夭）**:
```
affinity_gain = 40  # 固定
```

**入梦（狐仙/归女）**:
```
affinity_gain = 35  # 固定
```

**修炼（山翁）**:
```
success_rate = mining_level * 5 + foraging_level * 5
if random() < success_rate:
    affinity_gain = 40  # 成功
else:
    affinity_gain = 10  # 失败
```

### 3. 缘分等级判定

```
def get_affinity_level(affinity):
    if affinity >= 2500: return "eternal"
    elif affinity >= 1800: return "devoted"
    elif affinity >= 1000: return "trusting"
    elif affinity >= 400: return "curious"
    else: return "wary"
```

### 4. 每日衰减公式

```
if not interacted_today and not offered_today:
    if bonded:
        affinity = max(0, affinity - 15)
    elif courting:
        affinity = max(0, affinity - 10)
```

### 5. 缘分能力解锁判定

```
for ability in npc.abilities:
    if state.affinity >= ability.affinity_required and ability.id not in state.unlocked_abilities:
        unlock_ability(ability)
```

## Edge Cases

### 1. 发现系统边界情况

| 情况 | 处理方式 |
|------|----------|
| 多个条件同时满足 | 按发现链顺序处理，每次只推进一步 |
| 条件不满足 | 保持当前阶段 |
| 已到达 revealed | 不再检查发现条件 |

### 2. 供奉边界情况

| 情况 | 处理方式 |
|------|----------|
| 今日已供奉 | 提示"今日已供奉过了" |
| 本周次数已满 | 提示"本周供奉次数已满" |
| 物品不存在 | 提示"背包中没有此物品" |
| 品质无效 | 使用最低可用品质 |

### 3. 互动边界情况

| 情况 | 处理方式 |
|------|----------|
| 今日已互动 | 提示"今日已互动过了" |
| 冷却中 | 提示"需要再等X天" |
| 未达到 revealed | 提示"尚未与此仙灵建立联系" |

### 4. 求缘/结缘边界情况

| 情况 | 处理方式 |
|------|----------|
| 缘分不足 | 提示需要多少缘分 |
| 已有羁绊 | 提示"已与其他仙灵有缘分羁绊" |
| 材料不足 | 提示缺少什么材料 |
| 解除需要10000g | 提示需要金钱 |

### 5. 缘分衰减边界

| 情况 | 处理方式 |
|------|----------|
| 结缘后继续供奉 | 缘分可继续增加，上限3000 |
| 结缘后不互动 | 每日衰减15，最低0 |
| 求缘后不互动 | 每日衰减10 |

### 6. 多日不上线处理

当玩家多日不上线时：
```
1. 缘分衰减: 每日结算，n天不上线 = n × 15（或10）
2. 发现进度: 保持不变
3. 供奉次数: 重置
4. 互动状态: 重置
5. 能力解锁: 保持不变
```
