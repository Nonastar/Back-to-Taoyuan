# 任务系统 (Quest System)

> **状态**: Approved
> **Author**: Claude Code
> **Last Updated**: 2026-04-07
> **System ID**: P08
> **Implements Pillar**: 社交系统与NPC互动

## Overview

任务系统是游戏的核心引导和奖励系统，包含三大类任务：告示栏委托（每日随机生成）、特殊订单（每周挑战）和主线任务（引导玩家体验完整游戏内容）。任务系统与 NPC好感度、成就、物品等系统深度联动，为玩家提供目标感和成就感。

## Player Fantasy

任务系统给玩家带来**目标感和探索的动力**。玩家应该感受到：

- **每日的小目标** — 告示栏委托提供轻松的日常任务
- **每周的挑战** — 特殊订单带来更大难度的追求
- **长期的故事** — 主线任务引导玩家体验完整游戏内容
- **丰富的奖励** — 完成任务的金钱、物品和好感奖励

**Reference games**: Stardew Valley 的委托系统；原神的任务引导。

## Detailed Design

### Core Rules

#### 1. 任务类型概述

**三类任务**:

| 类型 | 来源 | 时限 | 难度 | 奖励 |
|------|------|------|------|------|
| **告示栏委托** | 每日随机生成 | 2天 | 简单 | 金钱+好感+物品 |
| **特殊订单** | 每周生成 | 7天 | 中等~极难 | 金钱+好感+物品 |
| **主线任务** | 固定列表 | 无 | 递进 | 金钱+好感+物品+剧情 |

#### 2. 告示栏委托

**生成规则**:
- 每日新生成 1-2 个委托
- 根据当前季节筛选物品类型
- 从 NPC 候选池随机分配

**任务类型**:

| 类型 | 描述 | NPC 池 | 奖励倍率 |
|------|------|--------|----------|
| **delivery** (送货) | 交付物品给 NPC | 陈伯、刘娘、林老、小满 | ×3 |
| **fishing** (钓鱼) | 钓到指定鱼类 | 秋月、陈伯、林老 | ×3 |
| **mining** (采矿) | 采集指定矿石 | 阿石、小满、陈伯 | ×2 |
| **gathering** (采集) | 采集指定物品 | 林老、刘娘、小满 | ×3 |

**告示栏委托模板**:

| 类型 | 目标物品 | 数量范围 | 季节限制 |
|------|----------|----------|----------|
| delivery | 青菜、萝卜、土豆 | 2-5 | 春季 |
| delivery | 稻米、西瓜、辣椒 | 2-5 | 夏季 |
| delivery | 南瓜、红薯 | 2-5 | 秋季 |
| delivery | 冬小麦、大蒜 | 2-5 | 冬季 |
| fishing | 鲫鱼、鲤鱼、草鱼 | 1-3 | 混合 |
| mining | 铜矿、铁矿、金矿 | 3-8 | 无 |
| gathering | 木材、柴火、竹子 | 3-10 | 混合 |
| gathering | 草药、人参 | 2-5 | 春/夏/秋 |

**奖励计算**:
```
money_reward = unit_price × quantity × reward_multiplier
friendship_reward = 3~5
```

#### 3. 特殊订单

**生成规则**:
- 每周第 7/14/21/28 天生成
- 按难度梯度分层：简单(7天)→普通(14天)→困难(21天)→极难(28天)
- 只能同时存在一个特殊订单

**特殊订单梯度**:

| 梯度 | 生成日 | 时限 | 典型目标 | 金钱奖励 |
|------|--------|------|----------|----------|
| 1 (简单) | 第7天 | 7天 | 铜矿15、鲫鱼8、青菜10 | 400-600 |
| 2 (普通) | 第14天 | 7天 | 铁矿15、鲶鱼5、药材15 | 800-1200 |
| 3 (困难) | 第21天 | 7天 | 南瓜10、金矿15、人参6 | 2000-2500 |
| 4 (极难) | 第28天 | 7天 | 金矿25、南瓜20、鲶鱼12 | 3500-4500 |

**特殊订单模板示例**:

| 名称 | 目标 | 数量 | 梯度 | 金钱 | 物品奖励 |
|------|------|------|------|------|----------|
| 铜矿采购 | 铜矿 | 15 | 1 | 600g | 铁矿×3 |
| 鲜鱼征集 | 鲫鱼 | 8 | 1 | 500g | 鱼饵×10 |
| 铁矿备料 | 铁矿 | 15 | 2 | 1200g | 木炭×10 |
| 丰收计划 | 南瓜 | 10 | 3 | 2000g | 优质肥料×5 |
| 矿石大征集 | 金矿 | 25 | 4 | 4000g | 金矿×10+翡翠×2 |

#### 5. 任务接取与提交

**任务接取限制**:
- 最大同时进行任务数: 3 (告示栏委托 + 特殊订单)
- 主线任务单独计算，可同时进行

**任务完成判定**:

| 类型 | 完成条件 |
|------|----------|
| delivery | 背包中有足够物品，提交时扣除 |
| fishing/mining/gathering | 自动追踪背包中物品数量，或手动收集 |
| special_order | 自动追踪，提交时扣除 |
| main_quest | 所有目标达成，提交时扣除 deliverItem |

**任务失败处理**:
- 超时未完成自动移除
- 已接取的告示栏委托超时后消失
- 特殊订单未接取也会过期

#### 6. 奖励系统

**奖励类型**:

| 奖励类型 | 说明 | 示例 |
|----------|------|------|
| moneyReward | 铜钱奖励 | 300-5000文 |
| friendshipReward | 好感度奖励 | NPC好感+20~30 |
| itemReward | 物品奖励 | 鱼饵×10、肥料×5 |

**奖励结构对比**:

| 任务类型 | 金钱范围 | 好感范围 | 物品奖励 |
|----------|----------|----------|----------|
| 告示栏委托 | 50-500 | +3~5/个NPC | 可选 |
| 特殊订单 | 400-4500 | +5~15/个NPC | 必有 |
| 主线任务 | 300-5000 | +20~30/个NPC | 必有 |

### States and Transitions

#### 任务状态

| 状态 | 描述 |
|------|------|
| **available** | 告示栏上可接取 |
| **active** | 已接取，进行中 |
| **completed** | 已完成（已提交） |
| **expired** | 超时失败 |

**告示栏委托状态转换**:
```
available → active: 玩家接取
available → expired: 每日刷新清除
active → completed: 玩家提交
active → expired: 超时移除
```

#### 主线任务状态

| 状态 | 描述 |
|------|------|
| **pending** | 未接取（显示但不激活） |
| **active** | 已接取，目标进行中 |
| **completed** | 已完成提交 |

**主线任务状态转换**:
```
pending → active: 玩家接取
active → completed: 所有目标达成并提交
completed → next_pending: 自动初始化下一个任务
```

## Formulas

### 1. 告示栏委托奖励公式

```
# 金钱奖励
money_reward = unit_price × quantity × reward_multiplier

# 示例：铜矿×5 × 3 = 150g
# 示例：鲫鱼×3 × 3 = 135g

# 好感奖励
friendship_reward = 3~5 (固定值)

# 奖励倍率表
reward_multiplier = {
    "delivery": 3,
    "fishing": 3,
    "mining": 2,
    "gathering": 3
}
```

### 2. 特殊订单奖励公式

```
# 金钱奖励（固定值，不按单价计算）
money_reward = 400~4500 (根据梯度)

# 梯度对应
tier_1: 400-600g
tier_2: 800-1200g
tier_3: 2000-2500g
tier_4: 3500-4500g

# 好感奖励
tier_friendship = [5, 8, 12, 15]  # 梯度1-4
```

### 3. 主线任务目标评估

```
# 收获作物
harvestCrops = total_crops_harvested >= target

# 钓鱼数量
catchFish = total_fish_caught >= target

# 矿洞层数
reachMineFloor = highest_mine_floor >= target

# 技能等级
skillLevel = skill[skillType].level >= target
allSkillsLevel = all(skill.level >= target)

# NPC好感
npcFriendship = npc[npcId].friendship >= threshold
npcAllFriendly = all(npc.friendship >= friendly_threshold)

# 交付物品
deliverItem = inventory.has(item_id, quantity)

# 其他
earnMoney = total_money_earned >= target
cookRecipes = total_recipes_cooked >= target
completeQuests = completed_quest_count >= target
completeBundles = completed_bundle_count >= target
ownAnimals = animal_count >= target
married = has_spouse
hasChild = child_count > 0
```

### 公式变量表

| 变量名 | 类型 | 范围 | 说明 |
|--------|------|------|------|
| `money_reward` | int | 50-5000 | 金钱奖励 |
| `friendship_reward` | int | 3-30 | 好感奖励 |
| `reward_multiplier` | float | 2-3 | 奖励倍率 |
| `quantity` | int | 1-30 | 目标数量 |
| `tier` | int | 1-4 | 特殊订单梯度 |

## Edge Cases

### 1. 任务接取边界情况

| 情况 | 处理方式 |
|------|----------|
| 任务数已达上限 | 提示"最多同时接取3个任务" |
| 任务不存在 | 提示"任务不存在" |
| 已接取的任务 | 告示栏不再显示 |

### 2. 任务提交边界情况

| 情况 | 处理方式 |
|------|----------|
| 物品不足 | 提示具体缺少什么 |
| 目标未完成 | 提示当前进度 |
| 物品刚好够 | 提交后物品归零 |

### 3. 特殊订单边界情况

| 情况 | 处理方式 |
|------|----------|
| 无可生成订单 | 跳过本次生成 |
| 未接取已过期 | 自动清除 |
| 已有特殊订单 | 不能同时接两个 |

### 4. 主线任务边界情况

| 情况 | 处理方式 |
|------|----------|
| 前置未完成 | 无法接取下一个 |
| 目标未达成 | 不能提交 |
| deliverItem 物品不足 | 提示物品不足 |

### 5. 每日刷新边界情况

| 情况 | 处理方式 |
|------|----------|
| 告示栏刷新 | 清空旧任务，重新生成 |
| 特殊订单刷新 | 第7/14/21/28天生成新订单 |
| 任务超时 | 从活跃列表移除，记录过期 |

### 6. 多日不上线处理

当玩家多日不上线时：
```
1. 告示栏委托: 每日刷新，上线时重新生成
2. 特殊订单: 计算累计过期，上线时可能已过期
3. 主线任务: 进度保持不变
4. 累计完成任务: 保持不变
```
