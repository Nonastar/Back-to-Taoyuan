## 1. 设计原则

### 1.1 统一面板结构

三个面板遵循一致的四层视觉层级：

```
┌─────────────────────────────────────────┐
│ [Header] 标题 + 关闭按钮           50px │
├─────────────────────────────────────────┤
│ [TabNav] Tab切换按钮栏               44px │
├─────────────────────────────────────────┤
│                                         │
│ [Content] 主内容区（可滚动）             │
│                                         │
│                                         │
├─────────────────────────────────────────┤
│ [Footer] 底部状态/操作栏（可选）     50px │
└─────────────────────────────────────────┘
```

**设计 Token 应用**:
- `PANEL_BG = Color(0.12, 0.12, 0.16, 0.95)` — 面板背景
- `PANEL_BORDER = Color(0.25, 0.25, 0.32, 1.0)` — 面板边框
- `PANEL_HEADER_HEIGHT = 50` — 标题栏高度
- `BUTTON_HEIGHT = 44` — 按钮高度
- `SPACE_16 = 16` — 内边距
- `RADIUS_MD = 8.0` — 圆角

### 1.2 响应式规则

| 参数 | 值 |
|------|-----|
| 面板最大宽度 | 900px |
| 面板最大高度 | 700px |
| 面板最小宽度 | 600px |
| 面板居中 | 水平+垂直居中 |
| 背景遮罩 | 半透明黑色 50% |

### 1.3 无障碍标准

- 文字最小字号: 14px
- 文字颜色对比度: WCAG AA (4.5:1)
- 键盘导航: 完全支持 Tab/方向键导航
- 手柄支持: 完整支持
- 色盲安全: 不使用纯红绿区分状态

### 2.3 交互规范

| 操作 | 鼠标 | 键盘 | 手柄 |
|------|------|------|------|
| 切换Tab | 点击 | 左/右方向键 | LB/RB |
| 选择仙灵 | 点击卡片 | 上/下选择, Enter确认 | 摇杆选择, A确认 |
| 返回列表 | 点击返回 | ESC 或 Backspace | B返回 |
| 供奉 | 点击供奉入口 | Tab导航到按钮 | Tab导航到按钮 |
| 选择物品 | 点击物品 | 方向键选择, Enter确认 | 摇杆选择 |
| 确认供奉 | 点击按钮 | Enter | A |
| 关闭面板 | 点击X | ESC | B |
| 每日互动 | 点击按钮 | Tab+Enter | Tab+A |

**键盘快捷键**:
- `K` — 打开仙灵面板
- `ESC` — 关闭面板/返回上一级
- `Tab` — 循环切换可交互元素
- `Enter` — 确认选择
- `1-7` — 数字键快速选择供奉物品

### 2.4 数据需求

| 数据源 | 访问方式 | 用途 |
|--------|----------|------|
| HiddenNpcSystem | 单例访问 | 仙灵状态、缘分值、能力列表 |
| InventorySystem | 单例访问 | 可供奉物品列表 |
| TimeSystem | 单例访问 | 显灵日判定 |
| NotificationManager | 单例访问 | 操作反馈飘窗 |

**关键API调用**:
```gdscript
# 获取所有仙灵
HiddenNpcSystem.get_all_hidden_npcs()

# 获取单个仙灵状态
HiddenNpcSystem.get_hidden_npc_state(npc_id)

# 获取缘分等级
HiddenNpcSystem.get_affinity_level(npc_id)

# 获取已解锁能力
HiddenNpcSystem.get_unlocked_abilities(npc_id)

# 执行供奉
HiddenNpcSystem.perform_offering(npc_id, item_id, quality)

# 执行独特互动
HiddenNpcSystem.perform_special_interaction(npc_id)

# 检查是否显灵日
HiddenNpcSystem.is_manifestation_day(npc_id)
```

### 2.5 状态映射

| UI状态 | 数据状态 | 显示内容 |
|--------|----------|----------|
| **unknown** | 发现阶段 | 灰色锁图标 + "???" + "未发现" |
| **rumor** | 传闻阶段 | 半透明头像 + 名称 + "传闻" |
| **glimpse** | 惊鸿阶段 | 半透明头像 + 名称 + "惊鸿一瞥" |
| **encounter** | 邂逅阶段 | 彩色头像 + 名称 + "邂逅" |
| **revealed** | 往来阶段 | 彩色头像 + 完整信息 + 操作按钮 |
| **courting** | 求缘中 | 粉色光晕 + "求缘中" 标签 |
| **bonded** | 已结缘 | 金色光晕 + "已结缘" 徽章 |

**菱形显示规则**:
- 每颗菱形 = 250缘分
- 最多12颗 (3000缘分)
- 已点亮的菱形用 `ACCENT_GOLD` 颜色
- 未点亮的菱形用 `TEXT_MUTED` 颜色

### 2.6 动画需求

| 动画 | 触发时机 | 时长 | 效果 |
|------|----------|------|------|
| 面板打开 | 打开面板 | 200ms | 从中心缩放0.9→1.0，透明度0→1 |
| 面板关闭 | 关闭面板 | 150ms | 缩放1.0→0.95，透明度1→0 |
| 仙灵选中 | 选择卡片 | 100ms | 边框高亮 + 轻微放大 |
| 供奉成功 | 确认供奉 | 400ms | 物品飞入动画 + 缘分数字弹出 |
| 缘分增加 | 供奉/互动后 | 300ms | 菱形逐个点亮动画 |
| 能力解锁 | 达到能力门槛 | 600ms | 金光闪烁 + 解锁提示 |
| 切换视图 | 列表↔详情 | 200ms | 横向滑动过渡 |

# 成就面板信号

- 动画使用Tween而非帧循环
