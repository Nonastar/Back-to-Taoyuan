## Prototype Report: Farming Core Loop

### Hypothesis
农场核心循环（耕地→播种→浇水→收获）的操作反馈及时，节奏令人满意，可以进入生产阶段。

### Approach

**构建内容:**
- 3×3 地块网格
- 4种工具切换（锄头/种子/浇水壶）
- 体力系统（100点，每次操作消耗3-5点）
- 作物生长状态机（wasteland → tilled → planted → growing → harvestable）
- 时间推进（空格键跳过一天）
- 视觉反馈（颜色变化、作物大小）

**开发时间:** 约1小时

**简化处理:**
- 使用纯色方块代替精灵图
- 跳过存档/音效
- 单一作物类型（番茄）
- 固定4天生长周期

### Result

| 操作 | 预期 | 实际 | 评估 |
|------|------|------|------|
| 耕地 | 即时反馈 | 即时变色 | ✅ |
| 播种 | 消耗种子 | 种子-1 | ✅ |
| 浇水 | 颜色变化 | 作物变红 | ✅ |
| 收获 | 返还种子+奖励 | 种子+1 | ✅ |
| 时间推进 | 作物生长 | 进度条+大小 | ✅ |

**观察:**
- 体力消耗感觉略快，建议降低到2-3点
- 4天生长周期略长，新手可能感到挫败
- 收获反馈不够明显，需要更强烈的视觉/音效
- 整体循环逻辑正确，可以正常工作

### Metrics

| 指标 | 数值 |
|------|------|
| 帧率 | 60fps (稳定) |
| 操作响应 | < 16ms |
| 内存占用 | < 10MB |
| 代码行数 | ~200行 |

### Recommendation: PROCEED

**理由:**
1. 核心循环逻辑验证通过
2. 状态机设计合理，易于扩展
3. 玩家操作反馈基本满足要求
4. 代码结构清晰，可重构为生产代码

**需要改进:**
- 体力消耗从5点降至2-3点
- 番茄生长周期从4天降至3天
- 增加收获时的粒子效果
- 考虑添加连击/效率加成

### If Proceeding

**架构要求:**
1. 提取 FarmPlot 为正式实体类
2. 状态机改为枚举而非字符串
3. 添加工具基类和数据驱动配置
4. 准备 Sprite 资源

**性能目标:**
- 支持 10×10 地块（100个）
- 帧率保持 60fps
- 存档加载 < 100ms

**范围调整:**
- 保留核心循环，移除简化版本
- 优先实现：耕地/播种/浇水/收获
- 延后实现：肥料/洒水器/巨型作物

**生产化工作量估计:** 3-5天
